## 3. 리뉴얼 방향 논의

### 3-1. 홈 탭 구조 개편

**이지원 (기획)**: 홈을 완전히 다시 생각해야 할 것 같아요.  
사용자가 앱을 열 때 딱 세 가지만 보여주면 어때요?

```
[홈 리뉴얼 제안 — 이지원]

① 지금 시장이 어때? (시장 요약 — 공포/탐욕, 지수, BTC 도미)
② 오늘 뭐가 움직이고 있어? (실시간 급등 TOP 5 — 전 시장 통합)
③ 지금 뭔 일이야? (속보 뉴스 3~5개)

이 세 블록만. 스크롤 없이 한 화면에.
```

**최유나 (디자인)**: 좋아요. 근데 "실시간 급등 TOP 5"는 지금 있잖아요.  
문제는 코인/국장/미장이 각각 따로 있어서 비교가 안 된다는 거예요.  
**전 시장 통합 Top 5**로 바꾸면, BTC가 오늘 +10% 뛰었을 때 삼성전자 +3%보다 위에 올 수 있죠. 이게 더 임팩트 있어요.

**이준혁 (PM)**: 동의해요. 탭별 무버가 아니라 전체 시장 무버. 그게 핵심이에요.  
카뱅 때 홈 피드 개편했을 때 봤는데, 사용자들은 카테고리보다 **"지금 제일 핫한 것"**에 반응해요.

**박서연 (FE)**: 기술적으로 그게 구현이 복잡하진 않아요. 코인/국장/미장 데이터를 하나의 배열로 합치고 등락률 기준 정렬하면 되는데... 근데 **국장 데이터 신뢰성 문제**가 먼저 해결돼야 해요. 잘못된 데이터가 1위에 뜨면 더 위험해요.

**김민준 (BE)**: 맞아요. 박서연 씨 말이 맞고. 저 사실 국장 데이터 소스 바꾸고 싶었어요.  
지금 allorigins → Yahoo Finance ^KS11 구조인데, **Stooq.com이 한국 지수도 지원**해요.  
`^SPX`처럼 `^WIG`... 아니 한국은 다른 심볼이지만. 아무튼 직접 CORS 없이 받을 수 있는 소스가 있어요. 제가 오늘 오후에 검증해볼게요.

**장성민 (QA)**: 민준 씨, 소스 바꾸기 전에 저한테 먼저 알려줘요. 테스트 케이스 같이 짜야 해요.

### 3-3. 종목-뉴스 연결 (맥락 단절 해소)

**이지원 (기획)**: 이게 저는 서비스의 본질과 연결된다고 봐요.  
지금 흐름이 `시세 보기 → 뉴스 보기`인데, 이게 분리되어 있어요.  
이상적인 흐름은 `급등 종목 발견 → 왜 올랐나 즉시 확인 → 매매 판단`이거든요.

```
[제안] 종목 클릭 시 차트 패널에 "관련 뉴스" 섹션 강화
- 지금: 차트 + 기본 정보만
- 개선: 차트 + 종목명 키워드 뉴스 TOP3 즉시 표시
- 뉴스 클릭 → 새 탭에서 원문 열기
```

**최유나 (디자인)**: 좋아요. 차트 패널 하단에 뉴스 3개짜리 섹션 넣으면 되겠어요.  
카드 형태로 제목 + 출처 + 시간만 보여주면 충분해요. 거기에 속보 배지.

**김민준 (BE)**: 지금 뉴스 API가 전체 뉴스를 한꺼번에 불러오는 구조예요.  
종목별 필터링은 프론트에서 키워드 매칭으로 하면 되는데... 문제는 CryptoCompare 뉴스는  
코인 뉴스만 있고, 국장/미장 종목 뉴스는 약해요.  
`NewsAPI.org`를 붙이면 종목별 검색이 가능한데 키 필요해요. 무료 플랜 100req/day.  
아니면 현재 뉴스 중 종목명 포함된 것만 필터링하는 방식으로 임시 처리할 수 있어요.

**이준혁 (PM)**: 일단 키워드 필터링 방식으로 MVP 먼저 해봐요. NewsAPI는 나중에 검토.

```
위험 1. KOSPI/KOSDAQ 지연
  - 현황: allorigins → Yahoo Finance, 최대 10분 딜레이

  - 개선: 갱신 시간 표시 강화 ("10초 전 기준" 등)

  - 개선: 가격 기준을 업비트 단일 소스로, CoinGecko는 시총/도미넌스만
```

토스증권도 "N초 전 기준" 표시해요.

**최유나 (디자인)**: 데이터 타임스탬프를 각 데이터 블록별로 작게 표시하면 어때요?

KOSPI 옆에 `14:31` 이런 식으로. 데이터 신선도를 시각화하는 거예요.

## 5. 이지원의 탭 구조 재설계 제안

> 이지원이 회의 끝나고 바로 정리해서 공유

### 현재 탭 구조의 문제
```
현재: [홈] [전체] [국내주식] [해외주식] [코인] [ETF]
문제:
- 홈과 전체가 목적이 비슷하게 느껴짐
- "전체"는 무엇을 전체로 보여주는 건지 불명확
- ETF가 뒤에 있어서 잘 안 쓰임
```

### 제안 탭 구조 A (현재 구조 개선)
```
[🏠 홈] [🇰🇷 국내] [🇺🇸 해외] [🪙 코인] [📊 ETF]
- 홈은 전체 요약 대시보드
- 전체 탭 제거 (홈이 그 역할)
- 탭 5개로 단순화
```

### 제안 탭 구조 B (목적 기반 재편)
```
[🔥 지금 핫한 것] [🇰🇷 국내] [🇺🇸 해외] [🪙 코인] [📊 ETF]
- 홈 탭을 "지금 핫한 것"으로 명확히 리브랜딩
- 전체 탭 제거
```

**이지원 코멘트**: B안이 서비스 정체성에 더 맞는데, 파급력이 커서 이준혁 오빠 최종 판단 필요.  
A안이 리스크 낮고 바로 실행 가능. 두 안 중 CEO분이 고르시면 됩니다.

## [결정 2] CDS (Coinbase Design System) 적용 방향

**결정: CDS 디자인 토큰 시스템을 기반으로 삼되, 한국 증권 색상 컨벤션으로 오버라이드**

**최유나 분석**:

> CDS는 국제 금융 표준 (green=상승, red=하락)을 쓴다.  
> 우리 타겟은 한국 투자자다. 빨강=상승, 파랑=하락이 몸에 배어 있다.  
> 색상을 국제 기준으로 바꾸면 사용자가 혼란을 겪고 신뢰가 깨진다.  
> **CDS의 구조와 토큰 체계는 가져오고, 색상값만 한국 컨벤션으로 오버라이드한다.**

```css
/* CDS 한국 증권 색상 오버라이드 */
:root {
  --cds-positive: #F04452;        /* 상승: 한국식 빨강 */
  --cds-negative: #1764ED;        /* 하락: 한국식 파랑 */
  --cds-positive-subtle: #FFF0F1;
  --cds-negative-subtle: #EDF4FF;
  --cds-surface: #FFFFFF;
  --cds-surface-alt: #F2F4F6;
  --cds-border: #E5E8EB;
  --cds-border-heavy: #C9CDD2;
  --cds-text: #191F28;
  --cds-text-muted: #6B7684;
  --cds-text-disabled: #B0B8C1;
  --cds-primary: rgb(0, 82, 255);  /* Coinbase Blue — CTA, 링크 */
  --cds-elevation-1: 0 2px 8px rgba(0,0,0,0.06);
  --cds-elevation-2: 0 8px 24px rgba(0,0,0,0.12);
}
```

**CDS에서 직접 차용할 것:**
- 타이포그래피 스케일 (label1, caption, body 등)
- 간격 시스템 (8px base)
- Border radius 체계 (12px 카드, 8px 버튼, 9999px pill)
- elevation1/2 그림자 체계
- DataCard 구조 → 종목 카드 리팩터링
- Table 컴포넌트 → WatchlistTable 리팩터링
- Breakpoints (phone 767 / tablet 1279 / desktop 1280+)

**CDS에서 오버라이드할 것:**
- fgPositive → #F04452 (한국 빨강)
- fgNegative → #1764ED (한국 파랑)
- bgPrimary 강조색 → 상황에 따라 기존 토스 스타일 유지
