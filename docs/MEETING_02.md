## 1. "평범한 서비스" 진단 — 냉정 분석

### 1-1. 홈 첫 5초 문제

**이지원 (Strategy)**: Airbnb에 있을 때 Jobs to be Done 분석을 엄청 많이 했는데요. 우리 서비스의 JTBD가 뭐냐면 — "나는 장 시작 전 3분 안에 오늘 뭐가 터질지 알고 싶다"예요. 그게 핵심 Job이에요.

근데 지금 홈 들어가면 뭐가 있어요? 먼저 공포/탐욕 지수 카드들 있고, 지수 미니칩 가로 스크롤 있고, TOP 8 무버 있고, 그 밑에 속보뉴스 있어요. 총 4개 구역이에요. 사용자가 어디를 먼저 봐야 해요?

**이준혁 (CPO)**: 아무것도 강조가 안 되니까 다 봐야 하죠.

**이지원 (Strategy)**: 맞아요. **5초 안에 "오늘의 핵심 시그널" 하나가 눈에 들어와야 하는데, 지금은 4개 섹션이 동등한 무게로 있어요.** a16z 포트폴리오 앱들 보면 "오늘의 숫자 하나"를 hero로 꺼내요. 우리는 그게 없어요.

**최유나 (Design)**: 제가 Coinbase에서 배운 게 있는데요. 데이터 앱에서 첫 시선은 왼쪽 상단이에요. 거기에 뭐가 있어야 하냐면 "지금 가장 긴급한 정보"여야 해요. 지금 우리 왼쪽 상단엔 제목 "지금 핫한 것"이랑 날짜가 있어요. 날짜는 사용자가 알아요. 몰라도 돼요.

**이준혁 (CPO)**: 정리하면 — 홈 첫 5초에 사용자가 뭘 해야 하는지 모른다. 동의하죠? 다 넘어가요.

### 1-2. 속보뉴스 중복 — 구조적 분석

**박서연 (FE)**: 코드 레벨에서 정확하게 보면요. 현재 `HomeDashboard.jsx`의 BLOCK 3가 `useAllNewsQuery()`로 전체 뉴스 불러와서 5개 슬라이스 보여줘요. `BreakingNewsPanel.jsx`의 기본 탭(🔴 속보)도 `useNewsAutoRefetch()`로 전체 뉴스 불러와서 20개 보여줘요. React Query 캐시가 같은 쿼리키면 공유되니까 API 호출 자체는 중복 아닌데...

**이지원 (Strategy)**: 사용자 경험은 중복이에요.

**박서연 (FE)**: 맞아요. 홈 탭 BLOCK 3 "속보 뉴스"랑 우측 패널 "🔴 속보" 탭이 같은 정보를 보여주는 두 개의 UI예요.

**최유나 (Design)**: 저 이게 왜 생겼는지는 알아요. 우측 패널이 `hidden lg:block`이잖아요. 모바일에서는 숨겨지니까, 모바일 사용자를 위해 홈 BLOCK 3에 뉴스를 넣은 거죠. 근데 문제는 데스크탑에서 두 개가 다 보인다는 거예요.

**김민준 (BE)**: 근데 솔직히 더 큰 문제는 뉴스 자체의 퀄리티예요. 지금 CryptoCompare RSS + allorigins 프록시로 긁어오는 구조인데, 국내 주식 관련 뉴스가 약해요. "코인 뉴스" 위주예요. 국장 투자자가 보면 뉴스 탭 열었을 때 코인 뉴스만 나오면 실망하죠.

**장성민 (QA)**: 실제로 어제 테스트에서 "국내" 탭 눌렀더니 KR 카테고리 뉴스가 2~3건밖에 없는 경우 있었어요. 이게 UX 문제가 아니라 데이터 문제예요.

**이준혁 (CPO)**: 그건 나중에 논의해요. 일단 지금 있는 게 얼마나 빠른지 정직하게 보는 거예요.

**장성민 (QA)**: 제가 실측해봤는데요. 업비트 기준 BTC 1분 5% 급등 시나리오로 테스트했을 때, Surge Banner 표시까지 실제 8~12초 걸렸어요. 이건 괜찮은 편이에요. 근데 국장은 못 했어요. 소스 딜레이 때문에.

## 2. 콘텐츠 재구조화 결정

### 2-1. 홈 BLOCK 3 운명 논의

**최유나 (Design)**: 자, 제가 디자인 관점에서 명확하게 정리할게요. 홈 BLOCK 3 "속보 뉴스"를 그냥 두면 두 가지 문제예요.

```
문제 A: 데스크탑에서 홈 BLOCK 3 뉴스 + 우측 패널 뉴스 = 중복
문제 B: 모바일에서는 우측 패널이 숨겨지니까 홈 BLOCK 3 뉴스가 필요
```

해결책 두 가지 있어요.

```
옵션 1: 홈 BLOCK 3 삭제
  - 데스크탑: 우측 패널로 통합
  - 모바일: 홈 하단에 "뉴스 더 보기" 버튼 → 뉴스 전용 시트

옵션 2: 홈 BLOCK 3를 완전히 다른 콘텐츠로 교체
  - "속보 뉴스" → "종목 연관 인사이트"로 변환
  - 지금 TOP 8 무버에 있는 종목들의 관련 뉴스만 보여줌
  - "삼성전자 +3.2% — 관련 뉴스: [반도체 수출 증가...]"
```

**이지원 (Strategy)**: 저는 옵션 2가 훨씬 강력하다고 봐요. 왜냐면 이게 바로 오전 회의에서 나온 "맥락 연결" 문제를 홈에서부터 해결하는 거거든요. 급등 종목 보여주면서 왜 올랐는지도 같이 보여주면, "아 이 앱 유용하다"가 되는 거예요.

**박서연 (FE)**: 기술적으로 옵션 2가 어떤 건지... 현재 `topMovers` 배열(TOP 8 무버)의 종목명으로 뉴스를 필터링하는 거죠? 예를 들면 삼성전자가 TOP 8에 있으면 뉴스 배열에서 "삼성전자" 포함된 것만 보여주는?

**이지원 (Strategy)**: 맞아요. 오전에 민준 씨가 말한 키워드 필터링 방식이에요.

**김민준 (BE)**: 프론트에서 하면 돼요. 근데 문제는 뉴스 소스가 영어권 위주라서 "Samsung Electronics" 같은 영문명이나 티커 심볼로 매칭해야 해요. 한글 "삼성전자"로 매칭이 안 될 수 있어요. 매칭 테이블이 필요해요.

**장성민 (QA)**: 매칭 안 될 때 빈 인사이트 카드가 나오면 더 나빠요. fallback이 중요해요.

**최유나 (Design)**: 맞아요. 매칭된 뉴스가 없으면 그냥 섹션 자체를 숨기면 돼요. 콘텐츠 없으면 UI도 없어야죠.

**이준혁 (CPO)**: 그럼 결론 내릴게요. BLOCK 3 방향은 — **"속보 뉴스" 완전 교체 → "지금 핫한 종목 인사이트"로 리네이밍 + 매칭 뉴스 표시, 없으면 섹션 숨김.** 이 방향으로 가요.

### 2-2. 우측 패널 강화 — "시장 인텔리전스 패널"로

**이지원 (Strategy)**: 우측 패널을 지금 "BreakingNewsPanel"이라고 부르는데, 이름부터 바꿔야 해요. 안에 뭐가 있어요? 뉴스(속보/국내/해외/코인) + 고래 알림이에요. 이걸 합치면 뭐냐면 **"시장 인텔리전스"**예요. 뉴스 + 고래 동향 + 나중엔 온체인까지. 이건 단순한 뉴스 피드가 아니에요.

**최유나 (Design)**: 비주얼 아이덴티티도 달라야 해요. 지금 패널이 그냥 흰 배경에 탭바인데, "인텔리전스 허브"처럼 보이게 하려면 조금 더 다크하거나 구분감 있는 처리가 필요해요.

**이준혁 (CPO)**: 그건 디자인 스프린트 주제로 빼요. 지금은 구조 얘기만.

**김민준 (BE)**: 저 구조 얘기 하나 더 하면, 지금 패널 탭이 `속보 / 국내 / 해외 / 코인 / 고래`인데요. "고래"가 맨 마지막 탭이에요. 탭 순서 자체가 중요도 순서처럼 느껴지는데, 실제로 고래 알림이 제일 희귀하고 임팩트 있는 정보예요. 탭 순서를 `고래 / 속보 / 국내 / 해외 / 코인` 으로 바꾸거나, 아니면 고래를 패널 상단에 상시 노출하는 걸 고려해봐야 해요.

**이지원 (Strategy)**: 저는 고래가 스페셜해야 한다고 봐요. 탭 안에 숨겨두는 게 아니라, 패널 상단에 "최근 고래 이벤트 1건" 미리보기를 항상 보여주고, 클릭하면 고래 탭으로 가는 구조. 이렇게 하면 "이 패널에 뭔가 다른 것도 있구나"를 사용자가 발견해요.

**장성민 (QA)**: 그 미리보기, 고래 이벤트 없을 때 빈 공간 나오면 이상해요. "오늘 아직 감지된 고래 없음" 같은 상태 메시지 처리 필요해요.

**이준혁 (CPO)**: 정리. 우측 패널 방향:
```
1. 명칭: "시장 인텔리전스 패널"로 개념 정의
2. 구조: 패널 상단에 고래 미리보기 핀(1건 항상 노출) → 탭 아래로 뉴스 피드
```

## 4. 차별화 포인트 3가지 도출

**이준혁 (CPO)**: 자, 이게 오늘 회의 메인 어젠다예요. 경쟁사가 못 하는 것 3가지 찾아야 해요. 다들 한 가지씩 제안해봐요.

**이지원 (Strategy)**: 저는 **전 시장 통합 인텔리전스**라고 봐요. 국장/미장/코인 세 시장을 하나의 맥락으로 연결해서 보여주는 서비스가 없어요. "오늘 미국 빅테크 나스닥 +2%, 삼성전자도 같이 올라, BTC도 위험자산 선호로 +3%" — 이 스토리를 한 화면에서 보여주는 데가 없어요.

**최유나 (Design)**: 저는 **인지 부하 최소화 금융 UX**예요. 지금 금융 앱들 보면 다 복잡해요. 인베스팅닷컴, 키움 HTS, 심지어 토스증권도 정보가 너무 많아요. 우리가 "핵심 신호 3개만 보여주는 깔끔한 앱"으로 포지션 잡으면 차별점이 돼요. Coinbase 메인 앱이 왜 성공했냐면 복잡한 거 다 빼고 심플하게 해서예요.

**김민준 (BE)**: 저는 **실시간 + 신뢰성의 동시 달성**이에요. 실시간 데이터 쓰는 앱이 많은데, 얼마나 빠른지 + 얼마나 정확한지를 동시에 보여주는 앱이 없어요. 우리가 데이터 타임스탬프, "지연 데이터" 배지, 소스 표기를 투명하게 하면 "이 앱은 솔직하다"는 신뢰가 생겨요. Stripe에서 배운 건데, B2B에서도 투명성이 신뢰의 핵심이에요.

**박서연 (FE)**: 저는 **30초 이내 완결되는 모바일 워크플로우**를 차별점으로 보고 싶어요. 앱 열기 → 오늘 핵심 시그널 확인 → 관련 뉴스 클릭 → 판단. 이 플로우를 30초 안에 완결하게 설계하면 "이 앱 진짜 빠르다"가 돼요. Vercel 대시보드가 왜 개발자들이 좋아하냐면 필요한 정보 즉시 나오거든요.

**장성민 (QA)**: 저는... 솔직히 기능 차별점보다 **"신호가 거짓말 안 하는 앱"**이 더 강력한 차별점이라고 봐요. 금융 앱에서 숫자 틀리면 끝이에요. 우리가 KOSPI 지연 투명하게 표시하고, 코인 가격 소스 명시하고, 고래 알림 오탐 없으면 — 그게 경쟁사가 못 하는 거예요. 다들 자기 데이터 정확하다고만 하고 실제로 틀리니까.

**이준혁 (CPO)**: 다 좋아요. 근데 5개를 3개로 줄여야 해요. 제가 Robinhood 다닐 때 차별점은 3개 이내여야 마케팅 메시지가 돼요.

*(5분간 토론)*

**이준혁 (CPO)**: 합의 도출할게요. 3개로 압축하면:

```
[차별화 포인트 최종 3가지 — 팀 합의]

1. 전 시장 통합 급등 인텔리전스
   "국장·미장·코인을 하나의 맥락으로"
   - 전 시장 통합 TOP 무버 (어디에도 없음)
   - 시장간 상관관계 시그널 (미래 기능)
   - 경쟁사: 단일 시장 전문화 vs 우리: 크로스 마켓

2. 급등 → 이유 → 판단 원스탑 플로우
   "시세 보고 뉴스 찾으러 다른 앱 열 필요 없음"
   - 급등 종목 탭하면 관련 뉴스 즉시 표시
   - 고래 움직임도 같은 컨텍스트에서
   - 경쟁사: 시세 앱과 뉴스 앱이 분리됨

3. 투명한 실시간성 (데이터 신뢰)
   "얼마나 빠른지, 얼마나 정확한지 숨기지 않음"
   - 데이터 타임스탬프 블록별 표시
   - 지연/실시간 상태 뱃지
   - 소스 명시 (업비트 실시간 / Yahoo 10분 지연 등)
   - 경쟁사: 틀려도 "실시간"이라 표기
```

**이지원 (Strategy)**: 1번이 핵심 USP(고유한 판매 제안)이고, 2번이 사용성 차별점, 3번이 신뢰 차별점이에요. 이 순서가 맞아요.

**최유나 (Design)**: 1번이 홈 화면 hero가 되어야 해요. "국장·미장·코인 전부 지금 핫한 것"을 첫 화면에서 느껴야 해요.
