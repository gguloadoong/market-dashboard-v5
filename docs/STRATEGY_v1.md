## 2. 네비게이션 메뉴명 최종안

### 현행 vs 개선안 비교

| 현행 | 개선안 | 변경 이유 |
|------|--------|---------|
| 홈 | **지금핫한** | "홈"은 기능을 설명하지 않음. 급등 종목 발굴이라는 목적을 탭명에 직접 표기 |
| 국장 | **국내** | "국장"은 증권 은어. 일반 투자자 접근성을 위해 표준어 사용 |
| 미장 | **미국** | 동일 이유. "미장"보다 "미국"이 검색·인식 측면에서 명확 |
| 코인 | **코인** | 유지. 이미 대중화된 용어이며 변경 시 혼란 발생 가능 |
| ETF | **ETF** | 유지. 금융 상품명이므로 변경 불필요 |

### 최종 탭 순서 및 이름

```
지금핫한  |  국내  |  미국  |  코인  |  ETF
```

**"지금핫한" 선택 이유:**
- "지금핫한"은 동사형 + 형용사형 조합으로 직관적 행동 유발
- "오늘장"은 장 마감 후 접속 시 어색함(오늘 장이 아직 안 열렸을 수 있음)
- "핫마켓"은 서비스명 후보와 겹쳐 탭명 차별화가 약해짐
- 토스증권의 "발견", 카카오페이증권의 "오늘의 추천"처럼 첫 탭을 큐레이션 개념으로 포지셔닝하는 트렌드와 일치

## 3. 홈 레이아웃 순서 최종안

### 현행 순서
```
① 급등/급락 TOP10
② 인사이트 (AI 시황 요약)
③ 시장 지수 (KOSPI / NASDAQ / BTC)
④ MarketSummaryCards
```

### 문제점 분석
- 사용자가 앱을 열었을 때 가장 먼저 알고 싶은 것은 "지금 시장이 오르고 있나 내리고 있나"임
- 현행 구조는 맥락(지수) 없이 결과(TOP10)를 먼저 보여줘서 "왜 이게 급등인지" 이해에 시간이 걸림

### 최적 순서 (최종안)

```
① 시장 지수 바 (MarketIndexBar)          ← 맥락 제공 (3초 안에 시장 방향 파악)
② 인사이트 카드 (AI 시황 요약 2~3줄)     ← "그래서 지금 어떻다"를 문장으로
③ MarketSummaryCards (섹터별 요약)       ← 섹터 흐름 파악
④ 급등/급락 TOP10                        ← 구체적 종목 발굴
```

**근거:**
- 정보 계층 원칙: 전체 → 부분 → 세부의 순서가 인지 부하를 낮춤
- 지수를 먼저 보면 TOP10의 등락률이 "많이 오른 건지 원래 장이 좋은 건지" 맥락이 생김
- 인사이트를 지수 바로 아래 배치하면 숫자(지수)와 해석(인사이트)이 한 시야에 들어와 효과 극대화
- MarketSummaryCards는 "어느 섹터가 강한가" 중간 단계 정보로 TOP10 진입 전 필터 역할

## 4. 코인 특화 정보 배치 전략

### 4-1. 비트도미넌스 & 김프 이동

**배치 위치:** 코인 탭 전용 인덱스 바 (기존 MarketIndexBar와 동일한 형태로 코인 탭에만 추가)

```
[ BTC 도미넌스: 54.2% ↑ ]  [ 김프: +2.3% ]  [ 전체 코인 시총: $2.1T ]
```

**이유:**
- 비트도미넌스와 김프는 국장·미장 투자자에게 의미가 없는 코인 전용 지표
- 홈이나 전체 화면에 노출 시 비코인 투자자에게 노이즈가 됨
- 코인 탭 진입 시 첫 줄에서 바로 보이면 코인 트레이더의 첫 판단 시간 단축

### 4-2. 공포탐욕지수 설명 방법

**문제:** 사용자가 "어떤 기준인지 모르겠다"고 피드백
**해결:** 인덱스 숫자 옆 `(?)` 아이콘 + 호버 툴팁

**툴팁 내용 초안:**
```
공포탐욕지수 (Fear & Greed Index)
Alternative.me 제공 | 0~100 점수

0~24  : 극단적 공포 — 과매도 구간, 저가 매수 기회 가능성
25~49 : 공포
50    : 중립
51~74 : 탐욕
75~100: 극단적 탐욕 — 과열 구간, 조정 가능성 경고

※ 시장 심리 보조 지표로만 활용. 투자 결정의 근거가 될 수 없음.
```

**추가 UX 개선:**
- 점수 숫자 색상 자동화: 0~24 파란색, 75~100 빨간색, 나머지 회색
- "극단적 공포" / "극단적 탐욕" 레이블을 점수 아래에 작게 표시

## 5. 국장 투자자 동향 (개인/외국인/기관)

### 5-1. 위치

**국장 탭 상단, 지수 바 바로 아래 전용 카드로 배치**

이유: 기관·외인 순매수/순매도는 국장 탭의 핵심 컨텍스트 정보임. 종목 카드 내 인라인 표기 시 비교가 어렵고, 전체 흐름을 한눈에 파악하기 어려움.

### 5-2. 표시 형태

**권장: 방향성 화살표 + 금액 + 미니 바 차트 조합**

```
┌─────────────────────────────────────────────┐
│  오늘 순매수/순매도 (KOSPI 기준)              │
│                                             │
│  개인    ▲ +3,420억   ████████░░░░         │
│  외국인  ▼ -1,850억   ░░░░████████         │
│  기관    ▼  -990억    ░░░░░░████░░         │
│                        [순매수] [순매도]     │
└─────────────────────────────────────────────┘
```

- 화살표만으로는 금액 규모를 모름 → 금액 병기 필수
- 바 차트는 세 주체 간 상대적 규모를 직관적으로 비교 가능
- 색상: 순매수 = 빨간색(상승), 순매도 = 파란색(하락) — 국내 증권 색상 컨벤션 준수
- 코스피 / 코스닥 탭 전환 가능 (기본: 코스피)

### 5-3. 데이터 소스 우선순위

| 우선순위 | 소스 | 방법 | 비용 |
|---------|------|------|------|
| 1순위 | **한국거래소 정보데이터시스템 (data.krx.co.kr)** | 비공식 API (JSON 직접 호출) | 무료 |
| 2순위 | **네이버 금융** | 크롤링 or 비공식 API | 무료 (불안정) |
| 3순위 | **한국투자증권 OpenAPI** | `FHKST01010100` 투자자별 매매동향 | 이미 사용 중인 키 활용 가능 |

**PM 권장: 3순위(한투 OpenAPI) 우선 시도**
- 이미 `VITE_KIS_APP_KEY` 환경변수가 세팅되어 있음
- `투자자별 매매동향` 엔드포인트에서 개인/외국인/기관 데이터 제공
- 갱신 주기: 장 중 1분, 장 마감 후 종가 고정

## 6. 뉴스 + 고래 알림 인사이트 전략

### 6-1. 뉴스를 투자 시그널로 활용하는 방법

**현행:** 뉴스 제목 + 출처 나열

**문제:** 뉴스를 읽어도 "그래서 내가 뭘 해야 하나"가 불명확

**개선 방향: 뉴스에 시그널 태그(Signal Tag) 부착**

각 뉴스 아이템에 자동 분류 레이블 추가:

| 레이블 | 의미 | 색상 |
|--------|------|------|
| 🔴 매수 시그널 | 실적 상향, 계약 체결, 정책 수혜 | 빨간 배지 |
| 🔵 매도 압력 | 실적 하향, 소송, 규제 리스크 | 파란 배지 |
| ⚡ 속보 | 발행 1시간 이내 | 주황 배지 |
| 🏦 중앙은행 | 금리·통화정책 | 회색 배지 |
| 🌍 거시경제 | 환율·무역·GDP | 회색 배지 |

분류 기준: 뉴스 제목 키워드 파싱 (기존 `cleanTitle()` 함수 확장)
- 매수 시그널 키워드: "계약", "수주", "상향", "돌파", "신고가", "흑자"
- 매도 압력 키워드: "하향", "손실", "소송", "조사", "적자", "위기"

**추가 기능: "뉴스 임팩트 스코어"**
- 뉴스 등장 후 30분 이내 해당 종목 등락률 변화를 트래킹
- 향후 데이터가 쌓이면 "이 뉴스 소스는 평균 +2.3% 반응" 같은 정보 제공 가능 (v4 이후)

### 6-2. 고래 알림 탭 밖 노출 UX 방안

**현행 문제:** 고래 탭을 누르지 않으면 고래 이벤트를 인지할 수 없음

**해결: 3-Layer 노출 구조**

#### Layer 1 — 탭 배지 (즉시 구현 가능)
```
뉴스  |  국내  |  미장  |  코인  |  🐋 고래 ②
```
- 고래 이벤트 발생 시 탭 이름 옆 숫자 배지 (읽지 않은 건수)
- 배지 색상: 10BTC+ 이벤트는 빨간색, 일반은 회색

#### Layer 2 — 우측 패널 상단 고정 배너 (핵심 개선)
```
┌────────────────────────────────────────┐
│ 🐋 [3분 전] Binance → 알 수 없는 지갑  │
│    5,230 BTC 이동 ($482M)  매도 압력 ↑ │
└────────────────────────────────────────┘
```
- 우측 뉴스 패널 최상단에 최신 고래 이벤트 1건 고정
- 클릭 시 고래 탭으로 이동
- 24시간 이내 이벤트 없으면 숨김 처리

#### Layer 3 — 토스트 알림 (선택적 활성화)
- 최초 진입 시 "대형 고래 이동 발생 시 알림을 받으시겠습니까?" 온보딩 팝업
- 동의 시: 50BTC+ 이벤트 발생할 때마다 화면 하단 토스트 3초 노출
- 비동의 시: Layer 1+2만 작동 (강요하지 않음)

### 6-3. 고래 알림 세분화 레이블 최종안

| 레이블 | 트리거 조건 | 투자 해석 | 표시 색상 |
|--------|-----------|---------|---------|
| 🏦 거래소 입금 | 알려진 거래소 지갑으로 이동 | 매도 압력 신호 — 단기 하락 가능 | 파란색 |
| 🏦 거래소 출금 | 알려진 거래소 지갑에서 이동 | HODLing 신호 — 매도 의사 없음 | 빨간색 |
| 🤝 OTC 가능성 | 지갑→지갑, 단일 대형 이동 | 기관 간 장외거래 가능성, 영향 중립 | 회색 |
| ❓ 미확인 이동 | 라벨 없는 지갑 간 이동 | 의도 불명, 관망 권장 | 회색 |
| 🇰🇷 업비트 대량체결 | 업비트 2천만원+ 단일 체결 | 국내 기관/세력 개입 가능성 | 주황색 |

**레이블 적용 로직 (기존 `getWhaleInsight()` 함수 확장):**
- Whale Alert에서 제공하는 `from.owner_type` / `to.owner_type` 필드로 "exchange" 여부 판단
- `from=exchange, to=unknown` → 거래소 출금
- `from=unknown, to=exchange` → 거래소 입금
- `from=unknown, to=unknown` → OTC 가능성 또는 미확인 이동 (금액으로 구분: 100BTC+ → OTC, 미만 → 미확인)

**표시 형태 예시:**
```
🐋 [방금 전]  🏦 거래소 입금 — 매도 압력
Binance 수신: 3,200 BTC ($295M)
Whale Alert · onchain verified

🐋 [12분 전]  🤝 OTC 가능성
Unknown → Unknown: 8,400 BTC ($774M)
단일 지갑, 기관 간 장외거래 추정

🐋 [28분 전]  🇰🇷 업비트 대량체결 — 국내 매수세
BTC/KRW 3.2억 시장가 매수
Upbit 실시간 체결
```
