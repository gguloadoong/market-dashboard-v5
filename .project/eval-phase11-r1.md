## 5개 항목 정량 채점

### 1. 기획 의도 부합 (가중치 x3)

**점수: 16/20**

- SignalFeed: 3시장 통합 시그널을 시간순으로 표시하는 기획 의도에 정확히 부합. 마켓별 배지(국내/미장/코인) 색상 분리, 방향성 이모지(bullish/bearish/neutral), 강도 바(1~5) 설계 완비.
- MarketTimeline: 시그널 + 속보 뉴스를 시간순 타임라인으로 병합하는 설계. 조건부 렌더링(속보 0건 시 숨김)은 합리적.
- getMatchConfidence: DIRECT/SECTOR/WEAK 3단계 신뢰도 분류 — 뉴스-종목 매칭 정밀도 향상.
- useInvestorSignals: 섹터 로테이션 감지(3단계 이상 순위 변동 시 시그널) 추가.
- 감점 이유: 시그널 엔진이 데이터 수집까지 3초+ 대기 → 초기 로딩 시 "시그널 수집 중..."만 보임. 첫 인상에서 빈 위젯 2개(투자 시그널 + 시그널 피드)가 눈에 띔.

### 2. 투자 인사이트 제공 (가중치 x2)

**점수: 15/20**

- 외국인/기관 연속 매수매도 시그널: 3일 연속 기준, 금액 포함 — 기관 투자자 동향 파악에 유용.
- 섹터 로테이션 감지: localStorage 기반 전일 순위 비교 — 자금 흐름 변화 포착.
- 거래량 이상치(95th percentile): 마켓별 상위 5% 거래량 종목 시그널.
- 뉴스-종목 매칭 신뢰도: DIRECT(종목명 직접 언급) vs SECTOR(섹터 키워드) 구분으로 노이즈 감소.
- 감점 이유: 시그널 데이터가 API 폴링 의존이라 실시간 스크린샷에서 실제 인사이트 내용 미확인. 섹터 로테이션은 localStorage 기반이라 첫 방문 시 prevRanks가 없어 시그널 미발화.

### 3. 차별화 시그널 경험 (가중치 x2)

**점수: 14/20**

- SignalFeed의 강도 바(5단계 시각화)는 경쟁 앱 대비 차별점.
- MarketTimeline의 세로 타임라인 UI(시간 + 색상 도트 + 이벤트)는 정보 밀도 높음.
- 3시장 통합 뷰는 국내/미장/코인을 한 피드에서 보는 독자적 경험.
- 감점 이유: 시그널 데이터 미수집 상태에서의 빈 화면 경험이 차별화를 무력화함. 타임라인도 속보 부재 시 완전 숨김 — 대안 콘텐츠(예: 오늘 예정 이벤트) 없음. 강도 바가 실제로 렌더되지 않음(Playwright 검증 0개).

### 4. 디자인/UX 직관성 (가중치 x2)

**점수: 16/20**

- 토스 디자인 시스템 준수: 색상 체계(#F04452 빨강, #3182F6 파랑, #FF9500 오렌지), 라운드 카드(rounded-2xl), 그림자(shadow-sm).
- 마켓 배지 색상 코드 일관성: 국내=빨강, 미장=파랑, 코인=오렌지 — 전체 앱에서 통일.
- StockBadge: min-h-[44px] 터치 타겟 준수, hover/active 피드백, confidence별 font-weight 차별.
- 빈 상태 처리: "시그널 수집 중..." 텍스트 — 로딩 스피너 없이 텍스트만이라 약간 단조롭지만 정보 전달은 충분.
- 감점 이유: 시그널 피드 + 투자 시그널 둘 다 빈 상태일 때 화면 상단 절반이 비어 보임. 스켈레톤 UI 없음.

### 5. 정확도/커버리지 (가중치 x3)

**점수: 15/20**

- newsAlias.js: KR_ALIASES 99개+, US_ALIASES 37개+, COIN_ALIASES 20개+ — 주요 종목 커버리지 충분.
- krStockList.js: 60개+ 종목, 14개 섹터(반도체/2차전지/바이오/자동차/플랫폼/금융/방산/철강/에너지/통신/건설/소비재/게임/엔터/유통) 매핑.
- getMatchConfidence: WEAK 매칭 제외 로직으로 거짓 양성 감소.
- isShortKeyword + BOUNDARY_FORCE: 영문 3~4자 키워드(arm, meta, coin 등) 단어 경계 매칭 — 거짓 양성 방지.
- 감점 이유: 섹터 로테이션의 localStorage 기반 순위 비교는 브라우저 초기화 시 유실. 시그널 엔진 자체의 정확도는 실제 시그널 발화 데이터 없이 코드 리뷰만으로 검증 한계.

## 주요 감점 요인 및 개선 권고

### 1. 시그널 빈 상태 문제 (가장 큰 감점 요인)

- 투자 시그널 위젯 + 시그널 피드가 동시에 "시그널 수집 중..." 표시 → 화면 상단 절반이 비어 보임
- 권고: 스켈레톤 UI 또는 직전 세션 캐시(localStorage) 표시. 혹은 시그널 미수집 시 위젯 축소/숨김 처리

### 2. 타임라인 조건부 미노출

- 속보 뉴스 0건 시 MarketTimeline이 완전 숨김 → 기획 의도인 "오늘의 타임라인"이 대부분 시간대에 미노출 가능
- 권고: 속보 없어도 시그널 이벤트만으로 타임라인 표시. 혹은 "오늘 예정 이벤트" 폴백 콘텐츠

### 3. 강도 바 미렌더

- Playwright 검증에서 `.w-1.h-3.rounded-full` 선택자 0개 — 시그널 피드가 빈 상태이므로 강도 바도 미렌더
- 실제 시그널 데이터가 있을 때의 시각적 경험을 검증하지 못함

### 4. 상세 패널 진입 미검증

- 뷰포트 밖 요소 클릭 실패 — fullPage 스크린샷에서는 보이지만 실제 인터랙션 불가
- 권고: 다음 라운드에서 scroll + 특정 요소 타겟 클릭으로 재시도

### 5. 섹터 로테이션 localStorage 의존

- 첫 방문 시 prevRanks 없음 → 시그널 미발화
- 권고: 초기 방문 시 기본 섹터 순위를 seed 데이터로 제공

## 결론

Phase 11의 코드 설계 품질은 높다. SignalFeed/MarketTimeline/getMatchConfidence/detectSectorRotation 모두 구조적으로 잘 설계되어 있고, 별칭 딕셔너리 커버리지도 충분하다. 그러나 실제 사용자 경험 관점에서 시그널 데이터 미수집 시의 빈 화면 처리가 가장 큰 약점이다. 시그널 엔진이 데이터를 수집하기 전까지 화면 상단의 핵심 위젯 2개가 비어 있는 상태는 "3시장 통합 시그널"이라는 Phase 11의 핵심 가치를 훼손한다.

**FAIL 판정. 빈 상태 UX 개선 후 Round 2 재평가 필요.**
