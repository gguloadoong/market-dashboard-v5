# 킥오프: 마켓레이더 v4.0 대개편

> 일시: 2026-03-18 | 참석: 이준혁(CPO), 이지원(Strategy), 최유나(Design), 박서연(FE), 김민준(BE), 장성민(QA)

---

## 개편 배경

지난 3주 커밋 18개 중 12개(67%)가 fix. 같은 영역(데이터 정합성, 뉴스 품질, UI 버그)에서 반복 패치.
근본 원인: 설계 전제가 "보여주기"에 맞춰져 있어, 사용자가 원하는 "판단 돕기"와 간극.

---

## 포지셔닝 재정의

**기존:** "급상승 종목을 누구보다 빠르게 캐치"
**신규:** "매수 결정 직전 5분을 함께하는 앱"

사용자가 실제로 원하는 것:
- 급등 목록 X → "왜 올랐는지 + 아직 기회 있는지"
- 정보 나열 X → 확신 또는 경고

---

## 수용 기준 (Acceptance Criteria)

| AC | 기준 | 우선순위 |
|----|------|----------|
| AC-1 | 앱 열면 "지금 매수 관심 종목" 1개 이상 근거와 함께 표시 | P0 |
| AC-2 | 종목 상세에서 "왜 지금 움직이는가" 1문장으로 답 | P0 |
| AC-3 | 국장/미장/코인 데이터 신뢰도 명시 (실시간/지연/마감) | P0 |
| AC-4 | 모든 종목 클릭 시 연관 종목 최소 3개 표시 | P1 |
| AC-5 | 뉴스 제목에 언론사명 노출 0건 | P1 |

## Phase 1: 데이터 신뢰성 (기반 공사)

| 작업 | 우선순위 | 완료 기준 |
|------|----------|-----------|
| Mock 초기값 0 보장 (자동화) | P0 | 앱 로드 후 SurgeBanner에 0% 종목 없음 |
| 데이터 신뢰도 레이블 표시 | P0 | 모든 탭에 실시간/지연/마감 표시 |
| 뉴스 소스 재설계 (종목별 동적 쿼리) | P0 | 임의 종목 클릭 시 관련 뉴스 70%+ 매칭 |
| API 에러 상태 UI 완성 | P1 | 모든 섹션 빈칸 없음 |

## Phase 2: UX 재구조화 (가치 재정의)

| 작업 | 우선순위 | 완료 기준 |
|------|----------|-----------|
| 홈 구조 개편 ("이유" 카드 최상단) | P0 | 앱 열면 3초 안에 "왜 움직이는지" 파악 |
| 선행 신호 섹션 (뉴스 발생→주가 미반응) | P1 | 신호 종목 1개 이상 표시 |
| 경제 이벤트 캘린더 | P1 | FOMC/CPI 등 주요 이벤트 표시 |
| 종목 상세 "왜 지금?" | P1 | 클릭 시 1문장 컨텍스트 |

---

## 합의사항

1. Phase 1 먼저 — 데이터 신뢰성 없이 어떤 UX도 의미 없음
2. 포지셔닝 전환 — 모든 기능 결정의 기준 = "매수 결정에 도움이 되는가?"
3. 뉴스 = 핵심 무기 — 종목별 뉴스 커버리지 차별화
4. "이유"가 "급등" 보다 중요 — 이유 없는 급등 목록은 가치 없음
