# 팀 회의록 — MEETING_03

## 현황 요약

- Phase 1~3 백로그 전체 완료 (커밋 기준)
- 이틀간 13개 파일, 155줄 변경
- 주요 완료: ETF 홈 노출, 빗썸 WS 연결, hantoo 날짜 버그 수정, 선행 신호 섹션, 경제 이벤트 캘린더

## 주요 논의

### 1. 클라이언트 직접 Yahoo 호출 문제 (김민준, 박서연)
- `fetchEtfPricesBatch`가 Yahoo Finance를 클라이언트에서 직접 호출
- 레이트리밋·IP 차단 리스크, CORS 에러 시 사용자 피드백 없음
- ETHU, BITX 등 소형 ETF 데이터 불안정의 근본 원인
- **결정**: `api/etf-prices.js` Vercel Function 추가하여 서버 프록시로 전환

### 2. 홈 정보 계층 과부하 (최유나, 이지원)
- 현재 홈 섹션 수: 핵심 시그널, 선행 신호, 경제 이벤트, 급등/급락, 탑뉴스, 고래 알림 = 6개+
- ADR-006 KPI "첫 액션까지 8초" 달성 여부 의심
- 배지 과다로 배지 의미 희석
- **결정**: 최유나 UX 리뷰 → 섹션 통합/축소 계획 수립

### 3. 선행 신호 장외 시간 이슈 (장성민)
- 장외 시간(16:00 이후)에 모든 종목이 "주가변화 1.5% 미만" 조건 충족
- 섹션이 항상 풀로 노출될 수 있음 → 사용자 오해 소지
- **결정**: 장 운영시간 외 섹션 숨김 또는 "장외" 배지 표시

### 4. newsAlias 번들 경고 (박서연)
- SignalSection.jsx dynamic import + EarlySignalSection.jsx static import 혼재
- Chunk 분리 불가 경고 빌드마다 발생
- 성능 영향 미미하나 구조적 노이즈
- **결정**: 다음 패치에서 static import로 통일
