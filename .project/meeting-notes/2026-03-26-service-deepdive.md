## 주요 발언 요약

### 이지원 (Strategy) — JTBD 기준 점검
- Job 1: 구조는 있으나 핵심 정보 3초 이내 파악 어려움. 스크롤 필요.
- Job 2: SurgeBanner 트리거 기준 내부적으로도 불투명.
- Job 3: 관련뉴스 알고리즘이 Sprint 6 이전까지 사실상 미구현(0건). 체크 표시는 있었음.
- 포지셔닝 우려: 아직 키움HTS에 가까움. 정보는 많은데 맥락이 없음.

### 최유나 (Designer) — 시각 계층
- 홈 9개 섹션 — 시선 유도 신호 없음.
- 다크모드 완료됐지만 정보 계층 다크 환경 검증 미완.
- 모바일 375px 장중 알림+배너 중첩 시나리오 미테스트.

### 박서연 (FE) — 성능
- CDS 번들 361KB (gzip 100KB) — TabbedChips, Button 2개만 사용. **심각한 번들 블로트**.
- `useStockAndRelatedNews` useMemo 의존성 체인 검증 필요.
- `fetchStockDirectNews` 7일 확장 후 rate limit 리스크 (장중 다중 패널 열기 시 429).

### 김민준 (BE) — API 신뢰성
- Yahoo v8 fallback 강화: ADR-011에서 P0 결정했으나 **미구현 상태**.
- `isFinancialNews` MARKET_ANCHOR_KW 추가 감사 필요.
- 한투 배치 병렬화 PR #164: 실제 응답시간 측정값 없음.
- RSS 소스 실패율: Vercel Analytics 실제 확인 안 됨.

### 장성민 (QA) — 크리티컬 3개
1. **크리티컬 1**: '투자' 제거 후 실제 금융 뉴스 차단 여부 테스트 케이스 없음.
2. **크리티컬 2**: EarlySignalSection 관련 util 클린업 범위 미검증.
3. **크리티컬 3**: P2-7 관심종목 뉴스 알림 — priceAlert.js와 통합 설계 필요, 이중 권한 요청 리스크.
- **배포 가능 여부**: 크리티컬 1 검증 후 조건부 가능.
