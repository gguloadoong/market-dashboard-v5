🎯 설계 철학

홈을 "구성(섹션)"이 아닌 "사건의 흐름(피드)"으로 본다. 사용자가 매번 들어올 때마다 "지금 일어나는 일"이 최신순으로 쌓여 있고, 가장 강한 시그널 하나가 맨 위 히어로로 부각. 섹션 개념이 없으므로 "어디서 뭐 찾아야 하지?" 문제 자체가 사라짐.

🔧 변경 범위 (크다)

  • home/index.jsx 완전 재작성 — 타임라인 피드 컨테이너
  • 신규 TimelineFeed.jsx — 시그널/뉴스/급등/섹터/이벤트 통합 스트림
  • 신규 HeroSignalPager.jsx — 스와이프 가능한 TOP3 히어로
  • 기존 섹션 컴포넌트(NotableMovers/AiDebate 등)는 제거 대신 "더 깊게 탐색" 뒤로 이동
  • HOME_CONTRACT 대폭 수정 필요 — ADR 추가 권장

⚠️ 리스크

  • 기존 섹션 레이아웃에 익숙한 사용자에게 급격한 변화 (학습 비용)
  • 타임라인 정렬 알고리즘 품질이 전체 UX를 결정 — 시그널 강도/시간 가중치 튜닝 필요
  • 구현 공수 6~8일 + architect/QA 반복 예상
  • 실패 시 롤백이 복잡 (feature flag로는 커버 가능하나 트래픽 전환 설계 필요)
