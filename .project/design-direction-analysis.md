# 마켓레이더 v5 홈 화면 디자인 방향 분석

> 시안 C "커맨드 센터" 레이아웃이 확정된 상태에서,
> 이 문서는 **시각 언어(Visual Language)** 를 정의한다.
> 코드 변경 없이 디자인 원칙과 가이드라인만 다룬다.

---

## 1. 디자인 원칙 5가지

### 원칙 1: 시그널이 시각적 주인공이다 (Signal First)

시세 숫자는 배경이다. 사용자의 시선이 가장 먼저 닿는 곳에 "왜 지금 이 종목인가"라는 시그널이 있어야 한다. 가격표 나열은 증권사 앱이 한다. 우리는 시그널을 판다.

- 히어로 시그널 카드는 화면에서 가장 큰 시각적 무게를 가진다
- 시세 숫자는 시그널의 증거로 존재한다 (독립 주인공이 아님)
- 시그널이 없는 시간대에도 "시장이 조용합니다"가 주인공이다 (빈 공간이 아님)

### 원칙 2: 3초 안에 시장 상태를 읽는다 (Glance Rule)

사용자는 아침 지하철에서 5분 안에 장 준비를 마쳐야 한다(ADR-006). 첫 3초 안에 "오늘 시장이 뜨거운가 차가운가"를 판단할 수 있어야 한다.

- Layer 1(커맨드 센터)은 뷰포트 상단 400px 안에 완결된다
- 온도바의 색상 하나로 시장 분위기가 전달된다
- 스크롤 없이 "강세/약세/중립" 판단이 끝나야 한다

### 원칙 3: 정보 밀도 > 장식 (Density over Decoration)

Bloomberg의 정보 밀도를 토스의 가독성으로 포장한다. 픽셀 하나가 정보를 전달하지 않으면 제거한다.

- 장식용 아이콘, 일러스트, 빈 여백은 없다
- 모든 색상은 의미를 갖는다 (빨강=상승, 파랑=하락, 초록=실시간, 주황=경고)
- 그라데이션, 그림자, 글로우는 정보 전달 목적일 때만 허용한다
- 여백은 "정보 그룹의 경계"를 나타낼 때만 사용한다

### 원칙 4: 한국 투자자의 근육 기억을 존중한다 (Convention Respect)

사용자가 10년간 키움HTS, 네이버증권, 업비트에서 체화한 시각 패턴을 깨지 않는다.

- 빨강=상승, 파랑=하락 (ADR-002 엄수)
- 가격은 오른쪽 정렬, 등락률은 가격 바로 옆
- 거래량 급증은 주황/노랑 계열
- 숫자는 tabular-nums로 자릿수 정렬

### 원칙 5: 쉬운 언어, 쉬운 시각 (Easy Signal)

"외국인 5일 연속 매수"는 "Institutional Net Buying Streak"보다 100배 낫다. 시각 언어도 마찬가지다. 차트 읽을 줄 모르는 사용자도 색상과 크기로 강약을 판단할 수 있어야 한다.

- 시그널 강도는 크기(면적)로 직관 전달
- easyLabel 텍스트가 시각적으로 가장 먼저 읽힌다
- 전문 지표 용어는 시각적으로 격하(작은 캡션, 낮은 대비)

## 2. 시각 계층 전략

### 2-1. 커맨드 센터 내부 시각 무게 배분

커맨드 센터는 4개 서브 위젯으로 구성된다. 각각의 시각적 무게(Visual Weight)를 명확히 구분해야 한다.

```
시각 무게 순서:
1. HeroSignalCard  ████████████  (40%) — 가장 크고, 가장 강한 색상
2. TemperatureBar  ██████████    (25%) — 전폭 프로그레스 바, 색상으로 분위기 지배
3. WatchlistMini   ████████      (25%) — 정보 밀도 높지만 시각적 강조 억제
4. EventStrip      ████          (10%) — 1줄 롤링, 시각적으로 가장 조용
```

### 2-2. Layer 1 (3초 GLANCE) 시각 강조 전략

**TemperatureBar (시장 온도)**
- 전폭 프로그레스 바가 커맨드 센터의 "천장" 역할
- 바의 채움 색상이 곧 시장 분위기: 빨강 계열=강세, 파랑 계열=약세
- 지수 미니카드(KOSPI, S&P, BTC)는 바 아래에 수평 배치, 작고 담백하게
- 공포탐욕 점수는 바 우측에 인라인 (별도 위젯 아님)

**HeroSignalCard (히어로 시그널)**
- 커맨드 센터에서 가장 큰 면적 차지 (md:col-span-3)
- 시그널 방향에 따라 카드 좌측 보더 3px 색상 적용 (상승=빨강, 하락=파랑)
- easyLabel이 카드 내 가장 큰 텍스트 (text-title 이상)
- 종목명 + 시그널 타입 배지가 상단, "차트 보기" CTA가 우하단
- 시그널이 강할수록 배경 tint가 진해짐 (bg-positive/bg-negative 투명도 조절)

**WatchlistMini (관심종목 컴팩트)**
- 히어로 시그널 옆에 배치 (md:col-span-2), 정보 전달 목적
- 시각적 강조 최소화: 보더 1px, 배경 bg-surface, 작은 폰트
- 가격과 등락률만 표시, 차트 미니라인은 선택적
- "내 종목에도 시그널 있으면" 종목명 옆에 작은 도트 표시 (색상만, 텍스트 없음)

**EventStrip (이벤트 롤링)**
- 커맨드 센터 하단 1줄, 배경 bg-alt로 시각적 분리
- 텍스트만, 아이콘 최소화
- FOMC, CPI 등 중요 이벤트는 fg-warning 색상으로 강조

### 2-3. Layer 2~3 시각 전략

**NotableMovers (주목할 종목 WHY 카드)**
- 수평 캐러셀, 카드 단위
- 각 카드에 "왜 이 종목인가" 1줄 요약이 시각적 주인공
- 등락률 배지가 카드 우상단에서 방향성 즉시 전달

**SignalSummaryWidget (시그널 보드)**
- 강세/약세 2열 분리, 각 열 상단에 카운트 배지
- 시그널 리스트는 정보 밀도 우선 (좁은 행간, 작은 폰트)

**Layer 3 탐색 영역**
- 시각적으로 가장 조용해야 함 (급등/급락, AI 토론, 뉴스)
- 탭 위젯으로 묶어 공간 절약
- 탭 전환으로 접근, 기본 상태에서 화면 점유 최소화

## 3. 색상 전략

### 3-1. 다크모드 기본 색상 체계

다크모드를 기본값으로 설계한다. 금융 대시보드는 장시간 모니터링이 목적이므로 눈의 피로를 줄이는 다크 배경이 적합하다. 현재 tokens.css에 이미 완전한 다크모드 변수가 정의되어 있다.

**배경 계층 (어두운 순)**

| 토큰 | 다크 값 | 용도 |
|------|--------|------|
| `--bg` | `#0F1117` | 앱 전체 배경 (가장 어둡고 넓은 면적) |
| `--bg-surface` | `#1A1D24` | 카드, 패널 배경 (배경 위에 떠 있는 느낌) |
| `--bg-alt` | `#1F2330` | 교차행, 서브 영역, EventStrip 배경 |
| `--bg-hover` | `#252A35` | 인터랙션 피드백 |

**핵심 원칙**: 배경 계층은 최대 4단계. 그 이상 쪼개면 시각적 노이즈가 된다.

**선/보더 계층**

| 토큰 | 다크 값 | 용도 |
|------|--------|------|
| `--border` | `#2A2F3C` | 카드 구분, 행 구분 (은은하게) |
| `--border-heavy` | `#3A4050` | 강조 구분, hover 보더 |

### 3-2. 시세 색상 (한국 컨벤션 — ADR-002 엄수)

| 상태 | 라이트모드 | 다크모드 | WCAG 대비 |
|------|----------|---------|-----------|
| 상승 (fg) | `#F04452` | `#FF6B77` | 6.74:1 (AA) |
| 하락 (fg) | `#1764ED` | `#5B9CF6` | 6.66:1 (AA) |
| 보합 (fg) | `#8B95A1` | `#6B7684` | — |
| 상승 (bg tint) | `#FFF0F1` | `rgba(255,107,119,0.12)` | — |
| 하락 (bg tint) | `#EDF4FF` | `rgba(91,156,246,0.12)` | — |

**절대 금지**: 상승/하락 색상을 국제 기준(초록/빨강)으로 바꾸는 것. 한국 사용자에게는 신뢰 파괴 행위다.

### 3-3. 시그널 강도별 색상 체계

시그널은 "방향(강세/약세)"과 "강도(강/중/약)" 두 축을 가진다. 방향은 빨강/파랑, 강도는 투명도(opacity)로 표현한다.

**강세 시그널 (빨강 계열)**

| 강도 | 배경 | 좌측 보더 | 텍스트 강조 |
|------|------|----------|-----------|
| 강 (strength >= 7) | `rgba(255,107,119,0.18)` | `3px solid var(--fg-positive)` | fg-positive, font-bold |
| 중 (strength 4~6) | `rgba(255,107,119,0.10)` | `2px solid var(--fg-positive)` | fg-positive |
| 약 (strength 1~3) | `transparent` | `1px solid var(--border)` | fg-muted |

**약세 시그널 (파랑 계열)**

| 강도 | 배경 | 좌측 보더 | 텍스트 강조 |
|------|------|----------|-----------|
| 강 (strength >= 7) | `rgba(91,156,246,0.18)` | `3px solid var(--fg-negative)` | fg-negative, font-bold |
| 중 (strength 4~6) | `rgba(91,156,246,0.10)` | `2px solid var(--fg-negative)` | fg-negative |
| 약 (strength 1~3) | `transparent` | `1px solid var(--border)` | fg-muted |

**강도 표현 규칙**:
- 강도는 색상의 "농도"로만 전달한다 (크기 변경 없음)
- 좌측 보더 두께가 강도의 1차 시각 단서 (1px/2px/3px)
- 배경 tint 투명도가 2차 시각 단서
- 아이콘, 스타, 느낌표 등 추가 장식으로 강도를 표현하지 않는다

### 3-4. 기능적 색상 팔레트 (전체)

| 용도 | 토큰 | 다크모드 값 | 사용처 |
|------|------|-----------|--------|
| 상승 | `--fg-positive` | `#FF6B77` | 가격 상승, 강세 시그널 |
| 하락 | `--fg-negative` | `#5B9CF6` | 가격 하락, 약세 시그널 |
| 경고 | `--fg-warning` | `#FFAA33` | 거래량 급증, 이벤트 주의 |
| 실시간 | `--live` | `#3DD68C` | WebSocket 연결 상태, 실시간 표시 |
| 링크/CTA | `--primary` | `#4E9BFF` | "차트 보기", 인터랙티브 요소 |
| 기본 텍스트 | `--fg` | `#E8EBF0` | 본문 |
| 보조 텍스트 | `--fg-muted` | `#8B95A1` | 캡션, 타임스탬프 |

**원칙**: 위 7가지 외에 새로운 색상을 도입하지 않는다. 모든 시각 정보는 이 팔레트 안에서 해결한다.

## 4. 타이포그래피 전략

### 4-1. 서체 스택

```
기본:    Pretendard, -apple-system, BlinkMacSystemFont, 'Apple SD Gothic Neo',
         'Noto Sans KR', sans-serif
숫자:    동일 스택 + font-variant-numeric: tabular-nums
코드/ID: 'SF Mono', 'Fira Code', 'Consolas', monospace (해시, 주소 등 한정)
```

Pretendard는 한글/영문/숫자 균형이 좋고, weight 100~900까지 지원한다. 별도 숫자 전용 폰트를 로드하지 않고 `tabular-nums`로 자릿수 정렬을 해결한다.

### 4-2. 크기 스케일 (tokens.css 기반)

| 토큰 | 크기 | 용도 | 무게 |
|------|------|------|------|
| `--text-display` | 2rem (28px) | 히어로 시그널 가격, 큰 숫자 | Bold (700) |
| `--text-title` | 1.125rem (18px~16px) | 섹션 제목, easyLabel 헤드라인 | Semibold (600) |
| `--text-body` | 0.9375rem (15px) | 본문, 시그널 설명, 뉴스 제목 | Regular (400) |
| `--text-label` | 0.875rem (14px) | 배지 텍스트, 종목명, 칩 | Medium (500) |
| `--text-caption` | 0.8125rem (13px) | 타임스탬프, 부연, 출처 | Regular (400) |
| `--text-micro` | 0.6875rem (11px) | 최소 텍스트, 면책 | Regular (400) |

### 4-3. 숫자 표시 규칙

**가격**
- `tabular-nums` 필수 (자릿수 정렬)
- 한국 주식: 원 단위, 천 단위 쉼표 (예: 82,000)
- 미국 주식: 달러 + 소수 2자리 (예: $142.50)
- 코인: 원화, 만 단위 축약 허용 (예: 1억 2,850만)
- 등락률: 부호 + 소수 1자리 + % (예: +1.2%, -3.4%)

**등락률 표시**
- 양수: `fg-positive` 색상, "+" 부호 명시
- 음수: `fg-negative` 색상, "-" 부호 명시
- 보합(+-0.05% 이내): `fg-neutral` 색상, 부호 없이 "0.0%"

### 4-4. 정보 계층별 텍스트 적용

```
히어로 시그널 카드:
  easyLabel 제목    → text-title, Semibold, fg
  종목명           → text-body, Medium, fg
  부연 설명        → text-body, Regular, fg-muted
  가격/등락률      → text-display, Bold, tabular-nums, fg-positive/negative
  "차트 보기"      → text-label, Medium, primary

관심종목 미니:
  종목명           → text-caption, Medium, fg
  가격             → text-caption, Regular, tabular-nums, fg
  등락률           → text-caption, Semibold, tabular-nums, fg-positive/negative

시그널 리스트 행:
  시그널 타입      → text-caption, 배지 형태, 배지 배경색
  easyLabel        → text-label, Regular, fg
  종목명           → text-caption, fg-muted
```

## 5. 컴포넌트 스타일 가이드라인

### 5-1. 카드

금융 대시보드에서 카드는 "정보 컨테이너"다. 시각적 화려함이 아니라 경계 인식이 목적이다.

**기본 카드 (`.sc`, `.section-card`)**
- 배경: `--bg-surface` (다크에서 `#1A1D24`)
- 보더: `1px solid var(--border)` — 그림자 대신 보더로 경계 표현
- 모서리: `--radius-lg` (14px) — 현재 `rounded-xl`과 동일
- 그림자: 기본 상태에서 없음. hover 시에만 `--elevation-1`

**이유**: 다크모드에서 그림자는 거의 보이지 않는다. 보더가 더 명확한 경계를 만든다. 라이트모드에서도 그림자 대신 보더를 1차 경계로 유지하면 모드 전환 시 시각적 일관성이 보장된다.

**히어로 시그널 카드**
- 좌측 보더 3px (시그널 방향 색상)
- 배경: 시그널 방향의 tint (`--bg-positive` 또는 `--bg-negative`)
- 나머지 3면 보더 없음 (좌측 보더가 충분한 시각 단서)
- 다른 카드보다 면적이 2~3배 커서 자연스럽게 시선 유도

**종목 카드 (NotableMovers WHY 카드)**
- 기본 카드 스타일 + 상단에 시그널 타입 배지
- hover: `--border-heavy` + `--elevation-1`

**금지 패턴**:
- 카드에 gradient 배경
- 카드에 glow/neon 효과
- 카드 내부에 또 다른 카드 (중첩 카드)
- 둥근 모서리 과도 (20px 이상)

### 5-2. 배지

배지는 "카테고리를 즉시 인식"시키는 마이크로 UI다. 텍스트 + 배경색으로만 구성한다.

**시그널 타입 배지**
- 형태: `rounded-lg`, `px-2 py-1`, `text-caption`, `font-semibold`
- 강세: `bg-positive` 배경 + `fg-positive` 텍스트
- 약세: `bg-negative` 배경 + `fg-negative` 텍스트
- 중립: `bg-alt` 배경 + `fg-muted` 텍스트

**시장 배지 (KR / US / COIN)**
- 형태: `rounded-full`, `px-1.5 py-0.5`, `text-micro`, `font-bold`
- KR: `bg-alt` + `fg-muted` (기본, 국장이 기본 맥락이므로 강조 불필요)
- US: `bg-alt` + `fg-muted` (동일)
- COIN: `bg-warning` + `fg-warning` (코인만 시각적으로 구분 — 변동성 주의 암시)

**임팩트 배지 (시그널 강도)**
- 배지 형태가 아니라 좌측 보더 두께로 표현 (5-3절 참조)
- 별도의 "HIGH" "LOW" 텍스트 배지는 사용하지 않는다 (시각 노이즈)

**거래량 배지 (`.badge-vol`)**
- 현재 스타일 유지: `rounded-full`, `bg-warning`, `fg-warning`, `text-micro`
- "거래량 3배" 같은 숫자를 포함

### 5-3. 버튼 / CTA

금융 대시보드에서 CTA는 절제되어야 한다. 매매 버튼이 아니므로 과도한 강조는 부적절하다.

**주요 CTA ("차트 보기")**
- 텍스트 버튼 형태: `text-label`, `fg: primary`, 화살표 아이콘 (`->`)
- 배경 없음, 보더 없음 — 텍스트 + 색상만으로 클릭 가능성 표현
- hover: `underline` 또는 `primary-muted` 배경

**보조 CTA ("더 보기", "전체 보기")**
- `text-caption`, `fg-muted`, hover 시 `fg` 전환
- 현재 `.sc-more` 스타일과 동일

**칩 / 필터 (`.chip`)**
- 현재 스타일 유지
- 활성: `bg: fg`, `color: bg-surface` (반전)
- 비활성: `bg-surface` + `border`

### 5-4. 리스트

**관심종목 리스트 (WatchlistMini)**
- 컴팩트 행: 최대 4행, 행 높이 40px
- 3열 구성: 종목명 (좌) | 가격 (우정렬) | 등락률 배지 (최우측)
- 행 구분: `1px solid var(--border)` (마지막 행 제외)
- 시그널 있는 종목: 종목명 좌측에 작은 도트 (4px, fg-positive/negative)

**시그널 리스트 (SignalSummaryWidget)**
- 강세/약세 2열, 각 열 내부는 수직 리스트
- 행 구성: 시그널 타입 배지 | easyLabel | 종목명
- 좌측 보더로 강도 표현

**뉴스 피드 (NewsFeedWidget / 우측 패널)**
- 현재 `.news-row` 스타일 유지
- 제목 1줄 + 출처/시간 1줄
- 관련 종목 태그가 있으면 제목 아래에 칩 형태로 표시

### 5-5. 프로그레스 / 게이지

**시장 온도 바 (TemperatureBar)**
- 전폭 프로그레스 바, 높이 8px, `rounded-full`
- 배경: `--bg-alt`
- 채움: ZONE 색상 (강한 경계=#3182F6 ~ 강한 강세=#F04452)
- 채움 비율: 시장 온도 점수 → 0~100% 매핑
- 바 위에 텍스트 라벨 없음 (라벨은 바 좌측 또는 하단에 별도 표시)

**공포탐욕 인라인 표시**
- 별도 게이지가 아닌 숫자 + 텍스트 라벨
- "72 탐욕" 형태, 숫자는 getFgColor()로 색상 결정
- 국가별(KR/US/COIN) 이모지 접두사

**시그널 카운트 바 (시그널 보드)**
- 강세/약세/중립 3개 세그먼트 바
- 비율 표시 (예: 강세 63% | 약세 27% | 중립 10%)
- 각 세그먼트 색상: fg-positive / fg-negative / fg-neutral

## 6. "AI 슬롭 방지" 디자인 가이드

### 6-1. AI 슬롭이란

AI가 생성한 코드/디자인에서 반복적으로 나타나는 비기능적 장식 패턴. "있어 보이지만 아무것도 전달하지 않는" 시각 요소. 사용자는 이것을 "싸구려 AI 앱"으로 즉시 인식한다.

### 6-2. 피해야 할 패턴 5가지

**패턴 1: 의미 없는 그라데이션**
- 카드 배경에 `linear-gradient(135deg, #667eea, #764ba2)` 같은 보라-핑크 그라데이션
- 헤더에 다크-투-라이트 그라데이션 배경
- 판단 기준: "이 그라데이션을 단색으로 바꾸면 정보 손실이 있는가?" → 없으면 슬롭

**패턴 2: 장식용 글로우/블러**
- 카드 뒤에 `blur(40px)` 색상 블롭
- 버튼에 `box-shadow: 0 0 20px rgba(99,102,241,0.5)` 네온 글로우
- 배경에 떠다니는 원형 블러 오브젝트
- 판단 기준: "이 효과를 제거하면 사용성이 떨어지는가?" → 아니면 슬롭

**패턴 3: 과도한 border-radius + 글래스모피즘**
- 모든 요소에 `rounded-3xl` (24px+) 적용
- `backdrop-filter: blur(16px)` + 반투명 배경의 남용
- 판단 기준: 모달 오버레이 등 맥락적으로 필요한 곳 외에는 금지

**패턴 4: 아이콘/이모지 과잉**
- 모든 섹션 제목 앞에 이모지 (화면 전체가 이모지 파티)
- 장식용 SVG 일러스트레이션
- 판단 기준: "이 아이콘이 없으면 사용자가 이 영역의 기능을 모르는가?" → 아는데도 있으면 슬롭

**패턴 5: 무의미한 애니메이션**
- 페이지 로드 시 모든 카드가 `fadeInUp` (0.3초 딜레이)
- hover 시 `scale(1.05)` + `rotate(2deg)` 같은 과도한 변환
- 끊임없이 맥동하는 pulse 효과 (실시간 표시 외)
- 판단 기준: "이 애니메이션이 상태 변화를 전달하는가?" → 아니면 슬롭

### 6-3. 추구해야 할 패턴 5가지

**패턴 1: 기능적 색상만 사용**
- 모든 색상에 "왜 이 색상인가"를 1문장으로 설명할 수 있어야 한다
- 빨강=상승, 파랑=하락, 주황=경고, 초록=실시간, 회색=보조
- 장식 목적의 보라, 핑크, 민트, 인디고는 사용하지 않는다

**패턴 2: 정보 밀도를 여백으로 조절**
- 그룹 간 여백: `--space-6` (24px)
- 그룹 내 항목 간: `--space-3` (12px)
- 여백이 곧 정보 구조를 만든다 — 선이나 구분자보다 여백이 먼저

**패턴 3: 상태 변화에만 애니메이션**
- 가격 변동 시 flash 효과 (flashUp/flashDown) — 현재 구현 완료, 유지
- 시그널 신규 발생 시 좌측 보더 pulse 1회 — 상태 변화 알림 목적
- 데이터 로딩 중 스켈레톤 shimmer — 로딩 상태 전달 목적
- 그 외 모든 애니메이션은 `transition-colors duration-150` 정도의 미세한 전환만

**패턴 4: 타이포그래피로 계층 표현**
- 색상이 아니라 크기와 굵기로 1차 계층 표현
- 가장 중요한 정보 = 가장 큰 텍스트 (text-display 또는 text-title)
- 보조 정보 = 작은 텍스트 + 낮은 대비 (text-caption + fg-muted)

**패턴 5: 데이터가 말하게 한다**
- "시장이 뜨겁습니다!" 같은 텍스트 메시지보다 온도 바의 빨간색이 먼저 보인다
- "+5.2%"라는 숫자 자체가 성과를 말한다 (별도 "수익중!" 텍스트 불필요)
- 빈 상태에서도 "데이터가 없다"는 것 자체가 정보다 ("시장이 조용합니다")

### 6-4. AI 슬롭 자가 평가 체크리스트

코드 리뷰 또는 디자인 검토 시 아래 질문에 하나라도 "예"이면 수정 대상:

```
[ ] 이 그라데이션을 단색으로 바꿔도 정보 손실이 없는가?
[ ] 이 그림자/글로우를 제거해도 UI가 정상 작동하는가?
[ ] 이 애니메이션이 상태 변화와 무관한가?
[ ] 이 아이콘/이모지를 제거해도 사용자가 기능을 이해하는가?
[ ] 이 색상이 7가지 기능 팔레트(상승/하락/경고/실시간/CTA/기본/보조) 밖인가?
[ ] 이 border-radius가 16px를 초과하는가? (pill 형태 칩/배지 제외)
[ ] 이 요소를 제거하면 정보 밀도가 올라가는가?
```

## 7. 종합: 커맨드 센터 디자인 비전

### 한 문장 요약

> **"Bloomberg의 정보 밀도를 토스의 가독성으로 포장하되, 시그널이 시각적 주인공인 다크모드 대시보드."**

### 시각 아이덴티티 키워드

| 키워드 | 의미 | 반대 |
|--------|------|------|
| 절제된 (Restrained) | 기능적 색상만, 장식 배제 | 화려한, 장식적 |
| 밀도 있는 (Dense) | 픽셀당 정보량 최대화 | 넉넉한, 여유로운 |
| 날카로운 (Sharp) | 시그널 강도가 즉시 인식됨 | 부드러운, 모호한 |
| 신뢰감 (Trustworthy) | 숫자가 정확하고, 색상이 일관됨 | 화려하지만 가벼운 |
| 한국적 (Korean Convention) | 빨강=상승, 파랑=하락, 원화 표기 | 글로벌 표준 |

### 현재 tokens.css와의 호환성

이 문서에서 제안하는 모든 색상, 타이포, 간격 값은 현재 `src/styles/tokens.css`에 이미 정의되어 있다. 새로운 CSS 변수를 추가할 필요 없이 기존 토큰을 조합하여 모든 시각 요소를 표현할 수 있다.

추가가 필요할 수 있는 토큰:

| 후보 토큰 | 값 | 용도 | 추가 판단 |
|----------|---|------|----------|
| `--signal-border-strong` | `3px` | 시그널 강도 "강" 좌측 보더 | 상수로도 충분, 토큰 미추가 가능 |
| `--signal-border-medium` | `2px` | 시그널 강도 "중" 좌측 보더 | 상동 |
| `--signal-tint-strong` | `0.18` | 시그널 배경 투명도 "강" | opacity 값을 토큰화할 필요성 낮음 |

결론: **현재 토큰 체계로 충분하다.** 구현 단계에서 필요하면 최소한으로 추가한다.
