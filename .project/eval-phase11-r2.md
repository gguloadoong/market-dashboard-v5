## 5개 항목 정량 채점

### 1. 기획 의도 부합 (가중치 x3)

**점수: 18/20** (R1: 16 -> R2: +2)

- SignalFeed 0건 숨김 처리로 빈 위젯 중복 문제 해소. 화면 레이아웃이 자연스러워짐.
- SignalSummaryWidget 대체 콘텐츠로 "시그널 수집 중" 상태에서도 유용한 정보 제공.
- 섹터 강세/약세 시그널 즉시 생성으로 첫 방문에도 시그널 데이터 존재 (강도 바 10개 확인).
- 타임라인 정상 렌더 — R1의 NOT_FOUND 해소.
- 감점: 부팅 시드 "주목할 움직임" 텍스트 매칭 실패 — 실제 문구가 다를 수 있으나 기능적으로 시그널 생성은 동작 중(강도 바 10개가 증거).

### 2. 투자 인사이트 제공 (가중치 x2)

**점수: 17/20** (R1: 15 -> R2: +2)

- 섹터 시그널 즉시 생성(+-2% 이상)으로 첫 방문에도 섹터 동향 인사이트 제공.
- 상세 페이지 뉴스 감성 바 확인 — "-1.0 억재 우세 (3건)"으로 시장 심리 즉시 파악 가능.
- WHY 뉴스 매칭 정상 — 비트코인 상세에서 관련 뉴스 즉시 표시.
- 연관 ETF 표시 — 비트코인 ETF(IBIT, FBTC) 가격/변동률 제공으로 투자 판단 보조.
- 감점: 외국인/기관 연속 매수매도 시그널은 국내주식 전용 — 코인/미장에서는 해당 인사이트 없음.

### 3. 차별화 시그널 경험 (가중치 x2)

**점수: 17/20** (R1: 14 -> R2: +3)

- 강도 바 10개 실제 렌더 확인 — R1에서 0개였던 것이 완전 해소. 5단계 시각화 차별점 동작.
- 빈 상태 대체 콘텐츠(감지 항목 태그)로 시그널 수집 전에도 기능 설명 제공 — 사용자에게 "곧 채워질 것"이라는 기대감 생성.
- 시그널 피드 0건 숨김으로 중복 빈 위젯 제거 — 화면 효율성 향상.
- 타임라인 정상 노출 — 시간순 이벤트 흐름 경험 가능.
- 감점: 시그널 피드가 숨겨진 상태에서 시그널 위젯만 단독 노출 — 피드와 위젯의 역할 분담이 사용자에게 즉시 명확하지 않을 수 있음.

### 4. 디자인/UX 직관성 (가중치 x2)

**점수: 18/20** (R1: 16 -> R2: +2)

- 빈 상태 처리 대폭 개선 — "시장 데이터를 분석 중" + 감지 항목 태그로 정보 밀도 유지.
- 0건 시그널 피드 숨김으로 불필요한 빈 공간 제거 — 레이아웃 밀도 향상.
- 상세 페이지 UX 확인: WHY 뉴스 → 투자자 동향 → 뉴스 감성 → 캔들 차트 → 연관 ETF 순서가 자연스러운 정보 탐색 플로우.
- 캔들 차트 타임프레임 선택(5분~1M)이 한 줄에 배치 — 모바일에서 한 번에 접근 가능.
- 감점: 상세 페이지 "투자자 동향" 섹션이 "국내주식만 제공" 안내만 있어 코인 상세에서는 빈 영역으로 느껴질 수 있음. 조건부 숨김 또는 대체 콘텐츠 권고.

### 5. 정확도/커버리지 (가중치 x3)

**점수: 17/20** (R1: 15 -> R2: +2)

- 섹터 시그널 즉시 생성으로 첫 방문 커버리지 확보 — R1의 localStorage 의존 문제 해소.
- 부팅 시드로 시그널 0건 상황 자체를 방지 — 변동폭 상위 3종목 즉시 시그널 생성.
- 뉴스-종목 매칭 정상 동작 확인 — 비트코인 상세에서 WHY 뉴스 + 감성 분석 모두 데이터 존재.
- 연관 ETF 가격 정확성 — IBIT $37.59, FBTC $57.78 실시간 반영.
- 감점: 부팅 시드 "주목할 움직임" 텍스트가 Playwright에서 매칭되지 않음 — 실제 문구 확인 필요. 기능적으로는 동작하나 검증 완전성 부족.

## 잔여 감점 요인 (R3 개선 대상)

### 1. 상세 페이지 "투자자 동향" 코인/미장 빈 영역 (-2점 영향)

- 비트코인 상세에서 "외인/기관/개인 순매수 데이터는 국내주식만 제공됩니다" 표시
- 코인/미장 종목에서는 이 섹션이 안내 문구만 차지 — 빈 공간
- 권고: 코인/미장일 때 조건부 숨김 또는 대체 데이터(거래량 추이, 고래 동향 등) 표시

### 2. 부팅 시드 텍스트 매칭 불일치 (-1점 영향)

- "주목할 움직임" 텍스트가 Playwright에서 0건 매칭 — 실제 구현 문구 확인 필요
- 기능적으로는 강도 바 10개가 렌더되므로 시그널 데이터는 존재
- 권고: 시드 시그널 문구를 명확하게 통일 (예: "부팅 시드" 태그 또는 일관된 라벨)

### 3. 시그널 피드 숨김 상태에서 위젯 단독 노출 (-1점 영향)

- SignalSummaryWidget만 보이고 SignalFeed는 0건 숨김 — 두 컴포넌트의 역할 관계가 불명확
- 권고: 시그널 위젯 내에서 "시그널 피드에서 상세 보기" 같은 안내 또는, 시그널 존재 시 피드도 함께 노출

### 4. 감성 바 "억재 우세" 오타 가능성 (-0.5점 영향)

- "억재 우세"가 "억세 우세"(bearish dominant)의 오타일 수 있음
- 권고: 감성 라벨 문구 검수 — "매도 우세" / "약세 우세" 등 명확한 한국어 표현

## 결론

R1 대비 26점 상승(183 -> 209)으로 대폭 개선되었다. 핵심 감점 요인이었던 빈 상태 UX가 해소되었고, 강도 바/타임라인/상세 페이지 모두 정상 동작을 확인했다. 그러나 209/240(87.1%)로 90% 기준에 7점 부족하다.

잔여 감점 요인은 주로 상세 페이지 조건부 콘텐츠(코인/미장 투자자 동향)와 마이너 문구 이슈로, 핵심 기능 결함이 아닌 UX 완성도 수준이다.

**FAIL 판정 (87.1%). 216점까지 7점 추가 필요. R3에서 상세 페이지 빈 영역 처리 + 문구 검수로 도달 가능.**
