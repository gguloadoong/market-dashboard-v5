| 투자자 동향 | 한투 API 외국인/기관 매수매도 | 중간 — 데이터는 있으나 시그널화 안 됨 |
| Fear & Greed | US(CNN) + Crypto(Alternative.me) | 중간 — 표시만, 전환 시점 알림 없음 |

**진단 결론**: 데이터 수집 인프라는 충분하다. 부족한 것은 **데이터를 시그널로 변환하는 해석 레이어**다.

## 영역 1: 투자 시그널 시스템

### 현재 상태

- `newsSignal.js`: 키워드 기반 속보 판단 + 중요도 점수 (URGENT_KW, IMPACT_BREAKING_KW, MACRO_KW)
- `newsTopicMap.js`: 14개 섹터 토픽맵으로 뉴스→섹터 추론
- `NotableMoversSection.jsx`: 변동폭 + 거래량 순위 + 뉴스 매칭 복합 점수
- `WhalePanel.jsx`: movementType별 방향성 배지 (HODLing/매도압력/차익거래)
- `priceAlert.js`: 가격/거래량 조건 알림 (PWA Push)
- `SurgeBanner.jsx`: 급등/급락 실시간 티커

### Gap 분석

| 보유 데이터 | 현재 활용 | 시그널로 전환 가능한 것 |
|------------|----------|----------------------|
| 고래 이동 방향 (거래소/지갑) | 배지 표시만 | 방향성 해석 (bullish/bearish) + 연속 패턴 |
| 뉴스 제목 키워드 | 임팩트 배지 (호재/악재) | 감성 점수 + 종목별 누적 감성 트렌드 |
| 거래량 데이터 | 급증 알림 (3배 기준) | 평소 대비 이상치 감지 + 시간대별 패턴 |
| 외국인/기관 매수매도 | 일별 데이터 표시 | 연속 매수/매도일 카운팅 + 추세 전환 감지 |
| Fear & Greed | 현재값 표시 | 구간 전환 시점 알림 (Extreme Fear→Fear 등) |
| 섹터별 등락률 | 단순 평균 | 전일 대비 자금 이동 방향 감지 |

### 구체적 실행 방안

#### S1. 시그널 엔진 코어 (`src/engine/signalEngine.js`)

모든 시그널을 통합 관리하는 중앙 엔진. 각 시그널 소스에서 이벤트를 수집하고, 가중치를 부여해 종합 스코어를 산출한다.

```
시그널 스키마:
{
  id: string,
  type: 'whale' | 'news' | 'volume' | 'investor' | 'feargreed' | 'sector',
  direction: 'bullish' | 'bearish' | 'neutral',
  strength: 1~5,           // 시그널 강도
  symbol: string | null,   // 특정 종목 또는 null(시장 전체)
  market: 'KR' | 'US' | 'COIN',
  title: string,           // 한 줄 요약 ("삼성전자 외국인 5일 연속 순매수")
  detail: string,          // 상세 설명
  timestamp: number,
  expiresAt: number,       // 시그널 유효 기간 (뉴스: 2시간, 고래: 30분, 투자자: 24시간)
  source: string,          // 원본 데이터 출처
}
```

**시그널 생성 규칙 (유료 API 없이 구현 가능한 것만)**:

| 시그널 | 트리거 조건 | 방향 | 강도 |
|--------|-----------|------|------|
| 거래량 폭발 | 20일 평균 대비 3배+ | neutral (방향은 가격으로) | 금액 비례 1~5 |
| 외국인 연속 매수 | 3일+ 연속 순매수 | bullish | 연속일수 비례 |
| 외국인 연속 매도 | 3일+ 연속 순매도 | bearish | 연속일수 비례 |
| 기관 연속 매수 | 3일+ 연속 순매수 | bullish | 연속일수 비례 |
| 뉴스 호재 집중 | 동일 종목 호재 뉴스 2건+ / 2시간 | bullish | 뉴스 수 비례 |
| 뉴스 악재 집중 | 동일 종목 악재 뉴스 2건+ / 2시간 | bearish | 뉴스 수 비례 |
| 고래 거래소 입금 연속 | 30분간 동일 코인 거래소 입금 2건+ | bearish | 금액 비례 |
| 고래 거래소 출금 연속 | 30분간 동일 코인 거래소 출금 2건+ | bullish | 금액 비례 |
| 스테이블코인 거래소 유입 | USDT/USDC 대량 거래소 입금 | bullish (매수 대기) | 금액 비례 |
| Fear→Extreme Fear 전환 | F&G 지수 구간 하락 | contrarian bullish | 구간 낙차 비례 |
| Greed→Extreme Greed 전환 | F&G 지수 구간 상승 | contrarian bearish | 구간 낙차 비례 |
| 섹터 로테이션 감지 | 전일 대비 섹터 순위 3단계+ 변동 | neutral | 변동폭 비례 |

#### S2. 시그널 대시보드 위젯 (`MarketPulseWidget` 확장)

현재 Market Pulse는 지수 6개 + 환율만 표시한다. 여기에 **오늘의 시그널 요약**을 추가한다.

```
[Market Pulse]
  코스피 2,650 +1.2%  |  S&P 5,200 -0.3%  |  BTC ₩131,500,000 +2.1%
  ─────────────────────────────────────────
  오늘의 시그널 (3건)
  🟢 삼성전자 — 외국인 5일 연속 순매수 (강도 4/5)
  🔴 BTC — 바이낸스 입금 30분간 3건 연속 (매도 압력)
  🟡 반도체 섹터 — 전일 대비 +3단계 상승 (로테이션 감지)
```

#### S3. 종목별 시그널 통합 뷰 (ChartSidePanel 확장)

종목 상세 패널의 "왜 지금?" 섹션을 시그널 기반으로 강화한다.

현재: 뉴스 제목 1줄
개선: 시그널 카드 스택 (최대 3개, 시간순)

```
[삼성전자 005930]
  왜 지금?
  ├─ 🟢 외국인 5일 연속 순매수 (3.2만주) — 2시간 전
  ├─ 🟢 "삼성전자 HBM3E 양산 본격화" — 한경 45분 전
  └─ 🟡 거래량 평소 대비 2.8배 — 현재
```

### 우선순위

| 순서 | 항목 | 난이도 | 기대 효과 |
|------|------|--------|----------|
| 1 | 시그널 엔진 코어 + 스키마 | 중 | 모든 시그널의 기반 |
| 2 | 외국인/기관 연속 매수매도 시그널 | 낮 | 기존 데이터 활용, 즉시 가치 |
| 3 | 거래량 이상치 감지 | 낮 | 기존 priceAlert 확장 |
| 4 | 고래 연속 패턴 감지 | 중 | WhalePanel 이벤트 버퍼링 필요 |
| 5 | F&G 전환 시점 알림 | 낮 | useFearGreed 확장 |
| 6 | 시그널 대시보드 위젯 | 중 | 사용자 체감 가치 최대 |
| 7 | 종목별 시그널 통합 뷰 | 중 | 탐색 루프 강화 |
| 8 | 섹터 로테이션 감지 | 중 | 전일 데이터 저장 필요 (Redis) |

## 영역 2: 뉴스 알고리즘 고도화

### 현재 상태

- `newsSignal.js`: 3단계 키워드 매칭 (URGENT_KW → IMPACT_BREAKING_KW → MACRO_KW)
- `newsAlias.js`: 종목별 alias 딕셔너리 + 단어경계 매칭
- `newsTopicMap.js`: 14개 섹터 토픽맵 (반도체, 2차전지, 금리, AI, 바이오 등)
- `getNewsImpact()`: 호재/악재 키워드 매칭 → 배지 표시
- `TopNewsSection.jsx`: 24시간 이내 + 시그널 점수 + 종목 매칭 기준 상위 5건

### Gap 분석

1. **카테고리 정확도**: `isFinancialNews` 블록 키워드 15개로 부동산/생활 뉴스 차단하지만, 경계선 기사(건설주 관련 부동산 뉴스)는 여전히 누수
2. **속보 판단**: 시간 + 키워드 조합이지만, 동일 이슈 후속 보도가 모두 속보로 분류됨
3. **뉴스-종목 연결**: alias 매칭은 직접 언급만 캐치. "반도체 수출 역대 최대" → 삼성전자/SK하이닉스 연결이 토픽맵으로 가능하나 정밀도 낮음
4. **클러스터링 없음**: 같은 이슈 후속 보도 5건이 각각 별개 뉴스로 표시
5. **감성 분석 부재**: 호재/악재 키워드 이진 분류만 있고, 감성 강도(매우 호재 vs 약한 호재) 구분 없음

### 구체적 실행 방안

#### N1. 뉴스 클러스터링 엔진 (`src/utils/newsCluster.js`)

**동일 이슈 후속 보도 묶기** — 유료 API 없이 구현 가능한 방법:

```
알고리즘:
1. 뉴스 제목에서 불용어 제거 (은/는/이/가/을/를/의/에/도/로 + 언론사명)
2. 남은 명사 토큰 추출 (한글 2자+ 연속 문자열)
3. 두 뉴스 간 토큰 Jaccard 유사도 계산
4. 유사도 > 0.4 + 시간차 < 6시간 → 같은 클러스터
5. 클러스터 내 가장 오래된 뉴스 = 원본, 나머지 = 후속 보도
6. UI: 원본 뉴스만 표시 + "관련 보도 N건" 접기 버튼

예시:
  원본: "삼성전자, HBM3E 양산 본격화…AI 수혜 기대" (한경, 09:15)
  후속: "삼성전자 HBM3E 양산 시작, SK하이닉스와 경쟁 본격화" (매경, 09:32)
  후속: "HBM3E 양산 삼성전자 주가 영향은" (연합, 10:05)
  → 클러스터 1건으로 묶임: "삼성전자 HBM3E 양산 (관련 보도 2건)"
```

#### N2. 감성 점수 고도화 (`newsSignal.js` 확장)

현재 이진 분류(호재/악재)를 **5단계 감성 점수**로 확장한다.

```
점수 체계:
  +2: 강한 호재 (실적 서프라이즈, 대규모 수주, 신약 승인)
  +1: 약한 호재 (목표가 상향, 신사업 진출, 제휴)
   0: 중립 (인사, 일반 공시, 시장 동향)
  -1: 약한 악재 (실적 하회, 소송, 경쟁 심화)
  -2: 강한 악재 (적자 전환, 리콜, 상장폐지 위험, 제재)

강도 키워드 사전 추가:
  STRONG_POSITIVE: ['사상 최대', '어닝 서프라이즈', '깜짝 실적', '흑자전환',
                     '대규모 수주', '신약 승인', 'fda 승인', '상장 확정']
  STRONG_NEGATIVE: ['적자전환', '상장폐지', '거래정지', '횡령', '분식회계',
                     '리콜 확대', '파산', '부도']
```

#### N3. 종목별 뉴스 감성 누적 트렌드

개별 뉴스 감성 점수를 종목 단위로 누적해서 **24시간 감성 트렌드**를 생성한다.

```
종목 감성 트렌드:
  삼성전자: 최근 24시간 뉴스 7건, 평균 감성 +1.3 (호재 우세)
  테슬라:   최근 24시간 뉴스 4건, 평균 감성 -0.8 (악재 우세)

UI 표현:
  종목 카드에 감성 미니 바 표시
  [████████░░] +1.3 호재 우세 (7건)
  [░░░░████░░] -0.8 악재 우세 (4건)
```

#### N4. 속보 중복 제거 + 시간 감쇠

```
현재 문제: "FOMC 금리 동결" 관련 뉴스 8건이 모두 속보로 표시
해결:
  1. 속보 판정 후 클러스터링 적용
  2. 클러스터 내 최초 1건만 속보 배지
  3. 속보 후 1시간 경과 시 "속보" → "주요 뉴스"로 자동 강등
  4. 2시간 경과 시 일반 뉴스로 전환
```

#### N5. 뉴스-종목 연결 정밀도 향상

```
현재: alias 직접 매칭 + 토픽맵 섹터 매칭 (OR 조건)
개선:
  1. 매칭 신뢰도 등급 도입:
     - DIRECT (종목명/심볼 직접 언급): 신뢰도 높음
     - SECTOR (토픽맵 섹터 추론): 신뢰도 중간
     - WEAK (단일 키워드 매칭): 신뢰도 낮음
  2. UI에 신뢰도 표시:
     - DIRECT: 종목 배지 굵게
     - SECTOR: 종목 배지 + "관련" 라벨
     - WEAK: 표시 안 함 (거짓 양성 방지)
  3. 토픽맵 확장: 14개 → 20개 섹터
     추가: 게임, 엔터, 유통/리테일, 통신, 방산, 우주항공
```

### 우선순위

| 순서 | 항목 | 난이도 | 기대 효과 |
|------|------|--------|----------|
| 1 | 감성 점수 5단계 확장 | 낮 | 시그널 엔진 입력값, 즉시 활용 |
| 2 | 뉴스 클러스터링 | 중 | 정보 피로 대폭 감소, UX 체감 큼 |
| 3 | 속보 중복 제거 + 시간 감쇠 | 낮 | 속보 탭 신뢰도 향상 |
| 4 | 매칭 신뢰도 등급 | 낮 | 거짓 양성 감소 |
| 5 | 토픽맵 20개 확장 | 낮 | 커버리지 확대 |
| 6 | 종목별 감성 누적 트렌드 | 중 | 시그널 엔진 + 종목 상세 강화 |

## 영역 3: 종목 발견 루프 (Discovery Loop)

### 현재 상태

현재 사용자 탐색 경로:

```
홈 (Market Pulse + 주목할 종목 + 뉴스)
  → 종목 클릭 → ChartSidePanel (차트 + "왜 지금?" + 관련 뉴스)
  → [막다른 길] 또는 뒤로가기
```

**문제**: ChartSidePanel에서 다음 종목으로 넘어가는 경로가 약하다.
`findRelatedItems`로 관련 종목 표시는 있지만, "왜 이 종목도 봐야 하는지" 맥락이 없다.

### Gap 분석

1. **종목→종목 연결 맥락 부재**: 관련 종목 리스트는 있지만 "왜 관련인지" 설명 없음
2. **섹터→종목 진입 경로 약함**: SectorMiniWidget → "섹터 탭에서 상세" 링크만 있고, 섹터에서 개별 종목으로 drill-down 불가
3. **뉴스→종목 역방향 경로 없음**: 뉴스 읽고 관련 종목으로 이동하는 경로가 약함 (배지 클릭 불가)
4. **고래→종목 연결 없음**: 고래 알림에서 해당 코인 상세로 이동하는 경로 없음
5. **시간축 탐색 불가**: "오늘 아침부터 뭐가 움직였지?" 시간순 타임라인 없음

### 구체적 실행 방안

#### D1. 종목 상세 "같이 보면 좋은 종목" 섹션 (ChartSidePanel 확장)

```
현재: relatedItems 리스트 (종목명 + 등락률)
개선: 연결 이유 태그 추가

[같이 보면 좋은 종목]
  SK하이닉스 +2.3% — "같은 반도체 섹터, 오늘 동반 상승"
  TSMC +1.1% — "공급망 연결, 미장 동반 반응"
  삼성SDI -0.8% — "삼성그룹주, 역방향 움직임"

연결 이유 생성 로직:
  1. 같은 섹터 → "같은 [섹터명] 섹터"
  2. 같은 그룹 → "[그룹명] 그룹주" (삼성, SK, LG, 현대 등)
  3. 뉴스 동시 언급 → "동일 뉴스 언급"
  4. 동반 등락 → "최근 동반 [상승/하락]"
```

#### D2. 뉴스 카드 종목 배지 클릭 → 종목 상세 이동

```
현재: TopNewsSection 종목 배지는 표시만 (클릭 불가)
개선: 배지 클릭 → onItemClick 호출 → ChartSidePanel 오픈

구현: StockBadge 컴포넌트에 onClick 핸들러 추가
  <StockBadge stock={s} onClick={() => onItemClick(matchedItem)} />
```

#### D3. 고래 알림 → 종목 상세 연결

```
현재: WhalePanel 이벤트 카드는 표시만
개선: 코인 고래 이벤트 클릭 → 해당 코인 ChartSidePanel 오픈

구현: WhalePanel 이벤트 카드에 onClick 추가
  - Upbit/Bithumb 이벤트: symbol로 coins 배열에서 매칭
  - Whale Alert/Blockchair: coin 심볼로 매칭
  - 매칭 실패 시 클릭 불가 (cursor-default)
```

#### D4. 섹터 drill-down (SectorRotation → 종목 리스트)

```
현재: SectorRotation은 섹터명 + 평균 등락률 바 차트
개선: 섹터 행 클릭 → 해당 섹터 종목 리스트 인라인 펼침

  [반도체 +2.1%] ← 클릭
    ├─ 삼성전자 +1.8%  거래량 TOP
    ├─ SK하이닉스 +3.2%  외국인 매수
    ├─ DB하이텍 +0.5%
    └─ 더보기 (전체 12종목)
```

#### D5. 시장 타임라인 위젯 (신규)

"오늘 시장에서 무슨 일이 있었는지"를 시간순으로 보여주는 위젯.

```
[오늘의 시장 타임라인]
  09:00  장 시작 — 코스피 2,648 출발
  09:15  삼성전자 HBM3E 양산 뉴스 → 반도체 섹터 +1.5%
  09:30  외국인 삼성전자 대량 매수 시작
  10:00  BTC 바이낸스 대량 출금 (2,000 BTC) → HODLing 시그널
  10:30  FOMC 의사록 공개 예고 → Fear & Greed 하락
  ...

데이터 소스:
  - 시그널 엔진 이벤트 (S1)
  - 뉴스 속보 (isBreakingNews)
  - 고래 알림 (WhalePanel 이벤트)
  - 시장 개장/마감 시각
```

#### D6. 순환 탐색 플로우 설계

최종 목표 플로우:

```
홈(시그널 요약)
  ↓ 시그널 클릭
종목 상세(차트 + 시그널 카드 + 뉴스)
  ↓ "같이 보면 좋은 종목" 클릭
다른 종목 상세
  ↓ 뉴스 클릭
뉴스 상세 → 다른 종목 배지 클릭
  ↓
또 다른 종목 상세
  ↓ 섹터 태그 클릭
섹터 drill-down → 종목 리스트
  ↓
홈으로 돌아오기 (또는 계속 순환)
```

**핵심 원칙**: 모든 정보 요소는 다음 탐색 목적지로의 링크를 가져야 한다.
현재 "표시만 하는" 요소를 "클릭 가능한 탐색 포인트"로 전환하는 것이 핵심이다.

### 우선순위

| 순서 | 항목 | 난이도 | 기대 효과 |
|------|------|--------|----------|
| 1 | 뉴스 배지 클릭 → 종목 이동 (D2) | 낮 | 즉시 구현, 탐색 경로 연결 |
| 2 | 고래 → 종목 연결 (D3) | 낮 | 기존 데이터, 클릭 핸들러만 추가 |
| 3 | 관련 종목 연결 이유 (D1) | 중 | "왜 관련인지" 맥락 제공 |
| 4 | 섹터 drill-down (D4) | 중 | 섹터→종목 탐색 경로 개통 |
| 5 | 시장 타임라인 (D5) | 높 | 시그널 엔진 의존, 차별화 높음 |

## 영역 4: 고래알림 시그널 해석

### 현재 상태

`WhalePanel.jsx` 분석 결과:

- **5개 소스**: Upbit WS (2000만원+), Bithumb WS, BTC 온체인 (10 BTC+), Whale Alert REST, Binance
- **방향성 배지**: movementType별 (거래소 입금=매도압력, 출금=HODLing, 거래소간=차익거래, 지갑간=OTC)
- **한국어 라벨링**: 거래소명 매핑 (EXCHANGE_NAME_MAP 17개), 이동 경로 한 줄 설명
- **금액 포맷**: 한국어 단위 (만/억/조) + USD 병기
- **이벤트 버스**: `whaleBus.js`로 BreakingNewsPanel에 최신 1건 공유

### Gap 분석

1. **단건 해석만 존재**: 각 이벤트를 개별적으로 해석. "30분간 거래소 입금 3건 연속"같은 **패턴 감지 없음**
2. **시그널 강도 없음**: 1억 이동과 100억 이동이 같은 크기의 카드로 표시
3. **스테이블코인 특수 로직 없음**: USDT/USDC 거래소 유입은 "매수 준비 자금"이라는 특수 의미가 있지만 일반 이동과 동일하게 처리
4. **지갑 유형 구분 없음**: 알려진 기관 지갑과 개인 지갑을 구분하지 않음 (Whale Alert API에 owner 필드가 있으나 활용 미비)
5. **누적 통계 없음**: "오늘 총 거래소 유입 vs 유출 비율" 같은 집계 데이터 없음

### 구체적 실행 방안

#### W1. 연속 패턴 감지 엔진 (`src/engine/whalePattern.js`)

```
이벤트 버퍼:
  - 최근 60분 이벤트를 메모리에 보관 (MAX_EVENTS 30 → 60으로 확장)
  - 30초마다 패턴 스캔

감지 패턴:
  1. 연속 거래소 입금 (30분, 같은 코인, 2건+)
     → "BTC 30분간 거래소 입금 3건 연속 — 강한 매도 압력"
     → direction: bearish, strength: 건수 x 금액 비례

  2. 연속 거래소 출금 (30분, 같은 코인, 2건+)
     → "ETH 30분간 거래소 출금 2건 — HODLing 확대"
     → direction: bullish, strength: 건수 x 금액 비례

  3. 대형 단건 ($10M+)
     → "BTC 150억원 바이낸스→콜드월렛 — 대형 HODLing"
     → direction: bullish/bearish, strength: 5 (최대)

  4. 스테이블코인 거래소 유입
     → "USDT 50억원 거래소 유입 — 매수 대기 자금 감지"
     → direction: bullish, strength: 금액 비례

  5. 양방향 동시 발생 (같은 코인, 같은 시간대에 입금+출금)
     → "BTC 거래소간 양방향 이동 — 차익거래 또는 리밸런싱"
     → direction: neutral, strength: 3
```

#### W2. 시그널 강도 시각화

```
현재: 모든 이벤트가 같은 크기 카드
개선: 금액 기반 3단계 시각화

  [일반] < 10억: 기본 카드 (현재와 동일)
  [주요] 10~50억: 테두리 강조 + "주요 이동" 배지
  [대형] 50억+: 카드 확대 + 배경 그라데이션 + 진동 애니메이션

  금액 계산:
  - Upbit/Bithumb: trade_price (KRW)
  - Whale Alert: amount_usd * 환율
  - Binance: tradeUsd * 환율
  - BTC 온체인: BTC 수량 * currentBtcKrwPrice
```

#### W3. 스테이블코인 특수 로직

```
Whale Alert 이벤트에서 symbol이 USDT/USDC/DAI/BUSD인 경우:
  - 거래소 입금: "매수 대기 자금 유입" (일반 코인과 정반대 해석)
  - 거래소 출금: "자금 이탈" (일반 코인과 정반대 해석)
  - 발행(mint): "신규 스테이블코인 발행 — 시장 유동성 증가 시그널"
  - 소각(burn): "스테이블코인 소각 — 유동성 축소 시그널"

UI: 스테이블코인 이벤트에 "💵" 아이콘 + 별도 배경색(연한 초록)
```

#### W4. 일간 고래 통계 요약

```
WhalePanel 상단에 오늘의 고래 요약 추가:

  [오늘의 고래 활동]
  총 이벤트 47건 | 거래소 유입 ₩1,230억 | 거래소 유출 ₩890억
  순유입 ₩340억 (매도 압력 우세)
  ────────
  [유입/유출 비율 바]
  ████████████░░░░░░░ 58% 유입

구현:
  - 이벤트 버퍼에서 금일 이벤트 집계
  - exchange_deposit 합산 vs exchange_withdrawal 합산
  - 비율 바 + 한 줄 해석
```

#### W5. 기관/알려진 지갑 라벨링

```
Whale Alert API의 owner 필드 활용:
  - owner_type: "exchange" / "unknown" 구분은 이미 됨
  - owner: 거래소명 (binance, coinbase 등) 은 EXCHANGE_NAME_MAP으로 처리 중

추가 라벨링 (정적 매핑):
  - 알려진 기관 지갑 주소 → 라벨 표시 (Grayscale, MicroStrategy, 테라폼랩스 등)
  - Blockchair 데이터에 포함된 label 필드 활용
  - 구현: KNOWN_WALLETS 딕셔너리 (주소 앞 8자리 → 라벨)

UI: "Grayscale 지갑 → 바이낸스 (기관 매도 신호)"
```

### 우선순위

| 순서 | 항목 | 난이도 | 기대 효과 |

### "여기서만 볼 수 있는 것" 후보

#### V1. 3시장 통합 시그널 피드 (핵심 차별화)

**경쟁사 누구도 국장+미장+코인의 시그널을 하나의 피드에서 보여주지 않는다.**

```
[시그널 피드] — 시간순
  10:15 🟢 삼성전자 — 외국인 5일 연속 매수 (국내)
  10:12 🔴 BTC — 바이낸스 대량 입금 (코인)
  10:05 🟡 반도체 섹터 — 3시장 동반 상승 (통합)
  09:50 🟢 NVDA — After-hours +3.2% (미장)
  09:30 🔴 USDT — 거래소 유입 급증, 매수 대기 (코인)

이것이 "왜 우리 서비스를 열어야 하는지"의 답이다.
토스에서는 삼성전자만, 업비트에서는 BTC만 보인다.
우리는 "지금 3개 시장에서 무슨 일이 벌어지고 있는지"를 한 눈에 보여준다.
```

#### V2. "WHY 카드" — 급등/급락 이유 1문장

```
현재: NotableMoversSection에 "왜 지금?" 뉴스 1줄 표시 (이미 구현)
확장: 뉴스가 없는 경우에도 이유를 추론

이유 생성 우선순위:
  1. 매칭 뉴스 있음 → 뉴스 제목 (현재 방식)
  2. 외국인/기관 대량 매수매도 → "외국인 N억 순매수"
  3. 거래량 급증 → "거래량 평소 대비 N배"
  4. 같은 섹터 동반 움직임 → "[섹터명] 섹터 전체 +N%"
  5. 위 모두 해당 없음 → "분석 중..." (표시 안 함이 아닌 투명한 상태 표시)

이것은 사용자가 스크린샷 찍어서 공유하고 싶은 콘텐츠다.
"삼성전자 +3.2% — HBM3E 양산 본격화 (외국인 5일 연속 매수)"
```

#### V3. 모닝 브리핑 (PWA 푸시)

```
매일 아침 8:50 (출근길) 푸시 알림:
  "오늘 시장: 코스피 야간선물 +0.3%, 나스닥 +1.2%, BTC 보합
   주목: 삼성전자(외국인 매수), 반도체(섹터 상승), BTC(고래 출금 증가)
```

#### V4. "공유하고 싶은" 시그널 카드

```
시그널 카드에 "공유" 버튼 추가:
  1. 카드를 이미지로 렌더링 (html2canvas 또는 Canvas API)
  2. 워터마크: "마켓레이더 — marketradar.app"
  3. Web Share API로 카카오톡/텔레그램/X 공유

공유 카드 디자인:
  ┌─────────────────────────────┐
  │ 🟢 삼성전자 +3.2%          │
  │ 외국인 5일 연속 순매수       │
  │ HBM3E 양산 본격화 (한경)    │
  │                             │
  │ 마켓레이더 marketradar.app  │
  └─────────────────────────────┘

이것이 입소문의 핵심 메커니즘이다.
"이거 어디서 봤어?" → 카드 하단 URL → 서비스 유입
```

#### V5. 모바일 PWA 경험 최적화

```
현재: 반응형 웹 (모바일 지원)
강화:
  1. 스와이프 제스처: 종목 카드 좌우 스와이프로 다음/이전
  2. Pull-to-refresh: 당겨서 새로고침 (네이티브 앱 느낌)
  3. 홈 화면 추가 유도 배너: "홈 화면에 추가하면 더 빠르게"
  4. 오프라인 fallback: Service Worker 캐시로 마지막 데이터 표시
  5. 하단 네비게이션 바 고정 (이미 MobileBottomNav 존재)

PWA manifest 강화:
  - shortcuts: 자주 가는 탭 바로가기 (관심종목, 시그널)
  - share_target: 종목 심볼 텍스트 공유받아 검색
```

## 기대 효과 종합

### 정량 목표

| 지표 | 현재 추정 | 목표 | 달성 방법 |
|------|----------|------|----------|
| 앱 열고 첫 인사이트까지 | ~8초 | 3초 | 시그널 대시보드 최상단 |
| 세션당 종목 조회 수 | ~3개 | 7개+ | 탐색 루프 연결 |
| 세션 체류 시간 | ~2분 | 5분+ | 순환 탐색 + 시그널 피드 |
| 공유 전환율 | 0% | 3%+ | 시그널 카드 공유 기능 |
| D1 리텐션 | 측정 불가 | 30%+ | 모닝 브리핑 + PWA |

### 차별화 요약

1. **국내 유일 3시장 통합 시그널**: 토스(국장만) + 업비트(코인만) + 키움(국장만) 어디에도 없는 통합 뷰
2. **"왜 움직이나" 자동 해석**: 시세 나열이 아닌 시그널 해석 — 데이터가 아니라 인사이트를 판다
3. **고래 움직임 방향성 해석**: 단순 알림이 아닌 "이게 매수 시그널인지 매도 시그널인지" 해석
4. **공유 가능한 시그널 카드**: 캡처→전송→유입의 바이럴 루프
