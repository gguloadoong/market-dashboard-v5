## 1. 경쟁사 홈 화면 분석

### 1-1. 토스증권 (Toss Securities)

**홈 구성**
- 내 투자 요약 (총자산, 수익률) -> 관심종목 -> 인기 종목 -> 뉴스
- 깔끔한 카드 UI, 정보 밀도 낮음, 여백 넉넉

**강점**
- 극도로 직관적인 온보딩 (첫 사용자도 3초 안에 이해)
- 수익률 색상이 화면 전체 톤을 결정 (심리적 앵커 효과)
- "주식 모으기" 등 독자 기능으로 습관 형성

**약점**
- 시그널/인텔리전스 전무 — "왜 올랐는지"는 알려주지 않음
- 코인/해외 분리 — 한 화면에서 3시장 볼 수 없음
- 뉴스와 종목 연결 없음 — 뉴스 탭 따로, 시세 탭 따로
- 조건 알림 설정 불가 (단순 급등 알림만)

**우리가 배울 것**: 정보 밀도와 여백의 균형. 첫 화면에서 "내 돈"이 보이는 심리적 안정감.
**우리가 이길 것**: 3시장 통합, 시그널 인텔리전스, "왜 지금" 맥락 연결.

### 1-2. 업비트 (Upbit)

**홈 구성**
- 실시간 시세 테이블 (원화마켓/BTC마켓/USDT마켓 탭)
- 상단: 검색 + 관심종목
- 하단: 투자내역 + 입출금

**강점**
- 실시간성 극대화 — WebSocket 기반 깜빡임 피드백
- 호가/체결 정보 밀도 높음 (트레이더 친화)
- 코인 특화 데이터: 펀딩비, 김프, 도미넌스

**약점**
- 코인 전용 — 주식 없음
- 시세 테이블 외에 인사이트 레이어 전무
- "왜 움직이는가"에 대한 답이 없음
- 뉴스 연동 부재

**우리가 배울 것**: 실시간 피드백의 즉각성 (가격 변동 시 셀 깜빡임/색상 전환).
**우리가 이길 것**: 주식+코인 통합, 뉴스-시세 맥락 연결, 시그널 해석 레이어.

### 1-3. TradingView

**홈 구성**
- 커스텀 대시보드 (사용자가 위젯 배치)
- 기본: 워치리스트 + 차트 + 뉴스 피드 + 경제 캘린더 + 스크리너
- 하단: 커뮤니티 아이디어 (소셜 트레이딩)

**강점**
- 위젯 자유 배치 — 파워유저의 성지
- 차트 기술 분석 도구 업계 최고
- 글로벌 모든 자산 클래스 커버
- 소셜 아이디어 (트레이더 의견 공유)

**약점**
- 초보자 진입 장벽 극히 높음
- 정보 과부하 — 커스터마이징 안 하면 혼란
- 모바일 경험 데스크톱 대비 크게 열악
- 한국어 지원 미흡, 한국 시장 특화 데이터 없음

**우리가 배울 것**: "스크리너" 개념 — 조건 기반 필터링의 강력함. 위젯 단위 사고.
**우리가 이길 것**: 모바일-퍼스트, 큐레이션된 시그널 (커스텀 불필요), 한국 시장 특화.

### 1-4. Robinhood

**홈 구성**
- 내 포트폴리오 그래프 (전체 자산 라인 차트, 기간 전환)
- "Today's Movers" 수평 카드 스크롤
- 뉴스 피드 (Snacks 브랜드 뉴스)
- 하단: 인기 종목 리스트

**강점**
- 포트폴리오 그래프가 감정을 지배 — 올랐으면 초록, 내렸으면 빨강 배경
- "Snacks" 뉴스 — 투자 뉴스를 쉬운 언어로 해석 (우리의 "우리만의 언어"와 동일 철학)
- 스와이프 제스처 자연스러움
- 매매까지 3탭 이내

**약점**
- 미국 전용 (한국 시장 없음)
- 시그널/알고리즘 분석 없음 — 단순 시세 + 뉴스
- 코인은 별도 섹션 (통합 아님)
- "왜 움직이는가" 맥락 부족

**우리가 배울 것**: "Today's Movers" 수평 카드 UX (한눈에 훑기). Snacks식 쉬운 뉴스 해석.
**우리가 이길 것**: 3시장 통합, AI 시그널, "조심할 이유"까지 제공.

## 2. "시그널 중심" 대시보드 UX 패턴

### 2-1. 정보 계층 원칙 (F-패턴 + 역피라미드)

금융 대시보드에서 사용자 시선 흐름은 **F-패턴**을 따른다:
1. **첫 수평 스캔** (상단 200px): 시장 전체 상태 파악 → "오늘 괜찮은 날인가?"
2. **두 번째 수평 스캔** (중단): 핵심 액션 아이템 → "지금 뭘 봐야 하는가?"
3. **수직 스캔** (좌측): 세부 정보 탐색 → "더 자세히 보자"

**역피라미드 적용**:
- **Layer 1 (Must-See, 3초)**: 시장 온도 + 가장 강한 시그널 1개
- **Layer 2 (Should-See, 10초)**: 주목 종목 + 관심종목 현황
- **Layer 3 (Nice-to-See, 30초)**: 급등/급락, 뉴스, 이벤트

### 2-2. 시그널 표현 패턴

**패턴 A: 카드 기반 (Robinhood / 현재 마켓레이더)**
- 시그널을 개별 카드로 나열
- 장점: 스캔하기 쉬움, 모바일 스와이프 친화
- 단점: 카드 수 많아지면 정보 과부하

**패턴 B: 대시보드 스코어보드 (Bloomberg LAUNCHPAD)**
- 전체 시장 상태를 단일 스코어/게이지로 요약 → 드릴다운
- 장점: 한눈에 전체 파악, "오늘 장이 어떤가" 1초 판단
- 단점: 개별 종목 정보 접근에 추가 클릭 필요

**패턴 C: 타임라인/피드 (Twitter/X 스타일)**
- 시그널을 시간순 피드로 표시
- 장점: "지금 무슨 일이 일어나고 있는가" 실시간 감각
- 단점: 중요도 구분 어려움, 놓치기 쉬움

**패턴 D: 히어로 + 리스트 하이브리드 (추천)**
- 가장 강한 시그널 1~2개를 히어로 카드로 크게 → 나머지 리스트
- 장점: 핵심 즉시 전달 + 나머지 스캔 가능
- 단점: 히어로 선정 알고리즘 품질에 의존

### 2-3. 액션 유도 패턴

**즉시 액션 (Immediate Action)**
- 시그널 카드에 "차트 보기" / "뉴스 보기" 인라인 CTA
- 탭 수: 시그널 발견 → 1탭으로 상세

**컨텍스트 전환 (Context Switch)**
- 시그널 클릭 → 차트 사이드패널 오픈 (현재 마켓레이더 방식)
- 페이지 이탈 없이 맥락 유지

**관심종목 즉시 추가**
- 시그널 카드에 별 아이콘 → 관심종목 추가
- 관심종목 추가 = 향후 알림 수신 동의

## 3. 모바일-퍼스트 금융 대시보드 베스트 프랙티스

### 3-1. 한 손 조작 원칙

- **Thumb Zone**: 화면 하단 1/3이 가장 접근성 높음
- **탭 네비게이션**: 하단 고정 (현재 마켓레이더 구현 완료)
- **스와이프 제스처**: 수평 카드 스크롤은 엄지 자연스러운 동선
- **Pull-to-Refresh**: 현재 구현 완료 — 유지

### 3-2. 정보 밀도 vs 가독성 밸런스

| 항목 | 모바일 | 데스크톱 |
|------|--------|---------|
| 동시 표시 카드 수 | 2~3개 | 4~6개 |
| 폰트 최소 크기 | 11px (본문), 10px (보조) | 12px (본문), 10px (보조) |
| 카드 간격 | 12px | 16px |
| 줄당 글자 수 | 25~35자 | 40~60자 |
| 시그널 카드 높이 | 80~100px | 64~80px |

### 3-3. 로딩 전략

- **스켈레톤 UI**: 이미 구현 (tabInitializing) — 유지
- **점진적 로딩**: 스크롤 아래 섹션은 IntersectionObserver로 지연 로딩
- **Redis 스냅샷 캐시**: 이미 구현 — 첫 로딩 <100ms
- **우선순위 로딩**: Layer 1 (시장 온도 + 시그널) 먼저 → Layer 2/3 비동기

### 3-4. 반응형 레이아웃 전략

**현재 구조**: `grid-cols-1 lg:grid-cols-[1fr_360px]`
- 데스크톱: 메인 콘텐츠 (좌) + BreakingNewsPanel (우)
- 모바일: 단일 컬럼 + 뉴스 탭 별도

**개선 방향**:
- 데스크톱 우측 패널 = "시장 인텔리전스 패널" (ADR-004) — 뉴스 + 고래 + 실시간 시그널 피드
- 모바일: 핵심 시그널이 홈에서 직접 노출 (우측 패널 콘텐츠 인라인화)
- Breakpoint: `lg:1024px` 기준 레이아웃 전환

### 3-5. 터치 인터랙션 패턴

| 제스처 | 동작 |
|--------|------|
| 탭 | 종목 상세 (ChartSidePanel) |
| 롱프레스 | 관심종목 추가/제거 퀵액션 |
| 수평 스와이프 | 카드 캐러셀 탐색 |
| 수직 스와이프 | 피드 스크롤 |
| Pull-down | 전체 새로고침 |

## 4. 핵심 인사이트 요약

### 4-1. 우리만의 포지션

```
토스증권 = "쉽고 예쁜데 정보 부족"
업비트   = "코인만"
키움HTS  = "정보 많은데 복잡하고 못생김"
TradingView = "프로 전용, 커스텀 필수"

마켓레이더 = "3시장 통합, 시그널이 주인공, 쉬운 언어, 모바일-퍼스트"
```

### 4-2. 홈 리뉴얼 핵심 원칙 5가지

1. **시그널이 주인공이다** — 시세 데이터는 배경, 시그널이 전경
2. **3초 안에 오늘의 시장을 읽는다** — Layer 1에서 시장 온도 + 최강 시그널
3. **"왜?"에 1탭으로 답한다** — 시그널 카드 → 차트+뉴스 즉시 연결
4. **내 종목이 항상 보인다** — 관심종목은 접히지 않는 영구 섹션
5. **쉬운 언어로 말한다** — easyLabel 체계 100% 적용, 전문용어 배제

### 4-3. Jobs to be Done 매핑

| Job | 현재 충족 | 리뉴얼 목표 |
|-----|----------|-----------|
| J1: 아침 5분 시장 파악 | B+ (섹션 10개로 산만) | A (Layer 1에서 3초 완결) |
| J2: 급등 종목 캐치 | A (NotableMovers) | A+ (히어로 시그널 + 푸시 연계) |
| J3: 내 포트폴리오 뉴스 확인 | B (관심종목이 6번째) | A (관심종목 + 시그널 통합) |
| J4: 고래 추적 | B (우측 패널 핀만) | A (시그널 통합, 모바일 접근성) |
