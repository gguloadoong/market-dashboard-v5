# 히어로 시그널 카드 + AI 종목토론 재설계 계획서

> 작성자: 이지원 (Head of Strategy) + 박서연 (FE)
> 작성일: 2026-04-07
> 상태: DRAFT — 팀 리뷰 후 확정

---

## 배경

대표님 피드백 원문:
> "히어로 시그널 카드가 컨텐츠 비어보이고 글씨만 큼. AI 종목토론도 UI/컨텐츠 구성 고민 필요."

**핵심 문제 진단:**

1. **히어로 시그널 카드**: 현재 `HeroSignalCard`는 TOP 3 시그널을 세로로 나열하지만, 각 카드에 종목명 + 시그널 설명 텍스트만 존재. 가격/변동률/거래량/sparkline 등 부가 정보가 전무하여 "글씨만 큰 빈 카드" 느낌.
2. **AI 종목토론**: 4종목 칩 선택 후 "살 이유/조심할 이유" 2줄 + 확신도 바가 전부. AI 분석의 근거 데이터가 없어 "AI가 분석했다"는 가치를 전달 못함.

**목업 vs 실구현 갭:**
- 목업(`mockup-final-light.html`)의 히어로에는 종목 로고 아이콘 + 가격 + 변동률 + 시그널 설명이 있으나, 실구현체에는 가격/변동률이 누락.
- 목업의 AI 토론도 현재 구현과 동일 수준 (2줄 요약 + 바) — 목업 자체가 정보 빈약.

## 1. 히어로 시그널 카드 재설계

### 현재 구조 분석

```
HeroSignalCard (useTopSignals(3))
├── 카드 1: [방향 배지] + [강도 도트 5개] + [시그널 타입 라벨]
│           [종목명 20px bold]
│           [시그널 설명 14px]
├── 카드 2: (동일)
└── 카드 3: (동일)
```

**문제점 상세:**
- 시그널 객체에 `symbol`, `name`, `market`, `direction`, `strength`, `type`, `meta` 필드가 있으나 카드에 `name`과 `type`만 사용
- 가격/변동률 데이터를 시그널 객체에서 직접 제공하지 않아 별도 조회 필요
- 종목 로고(`getLogoUrls`)를 사용하지 않음
- 빈 상태(시그널 없음)가 스켈레톤 + "시그널 수집 중..." 텍스트뿐 — 대안 컨텐츠 없음
- "edgeX", "EOS" 같은 소형 코인이 히어로에 올라오면 카드의 빈 느낌이 극대화

### 데이터 가용성

| 필드 | 시그널 객체 | 별도 조회 필요 | 비고 |
|------|-----------|-------------|------|
| 종목명 | `signal.name` | - | |
| 심볼 | `signal.symbol` | - | |
| 마켓 | `signal.market` | - | kr/us/crypto |
| 방향 | `signal.direction` | - | bullish/bearish/neutral |
| 강도 (1~5) | `signal.strength` | - | |
| 시그널 타입 | `signal.type` | - | TYPE_META에서 label 조회 |
| 메타 데이터 | `signal.meta` | - | 타입별 상이 (연속매수일수, 거래량배율, 고래이체금액 등) |
| 가격 | X | allItems에서 symbol 매칭 | |
| 변동률 | X | allItems에서 symbol 매칭 | |
| 거래량 | X | allItems에서 symbol 매칭 | |
| 스파크라인 데이터 | X | usePrices hook | 24h/7d 가격 배열 |
| 종목 로고 | X | `getLogoUrls(item)` | fallback 체인 있음 |

### A안: 정보 충실형 (Information-Rich Hero)

**컨셉**: 목업의 히어로를 충실히 구현하되 정보 밀도를 높임. 1개 큰 카드 + 2개 서브 카드.

```
[히어로 카드 — 1위 시그널 (큰 카드)]
┌─────────────────────────────────────────────┐
│ [로고] 삼성전자               외국인 연속매수   │
│        005930 · KR                          │
│                                             │
│ 82,400원                        +1.2%       │
│ ~~~~~~~~ sparkline (80x32) ~~~~~~~~         │
│                                             │
│ 강세 ●●●●○                                  │
│ "외국인 5일째 순매수, 3개월 최장 기록"          │
│                                             │
│ 외국인 누적: +2,400억  │  거래량: 평소 1.8배   │
└─────────────────────────────────────────────┘

[서브 카드 2위] [서브 카드 3위]  ← 가로 2열, 컴팩트
┌──────────────────┐ ┌──────────────────┐
│ [로고] SK하이닉스   │ │ [로고] 에코프로     │
│ 185,000원 +2.8%   │ │ 67,200원 -3.8%   │
│ 강세 ●●●○○        │ │ 약세 ●●●○○        │
│ 거래량 폭증        │ │ 심리 괴리          │
└──────────────────┘ └──────────────────┘
```

**장점:**
- 1위 시그널에 시각적 위계 부여 — "오늘의 핵심" 명확
- sparkline으로 추세 한눈에 확인 (Robinhood의 Today's Movers 패턴)
- meta 데이터(외국인 누적, 거래량 배율)로 시그널의 "왜"를 즉시 전달
- 종목 로고로 시각적 풍부함 확보

**단점:**
- 1위 카드가 세로 공간을 많이 차지 (모바일에서 스크롤 증가)
- sparkline 데이터 조회를 위한 추가 API 호출 필요
- 시그널이 1개뿐일 때 서브 카드 영역이 비어보일 수 있음

**구현 난이도:** 중 (sparkline 데이터 연결 + 가격 매칭 로직)

### B안: 균등 3카드 리치형 (Enriched Equal Cards)

**컨셉**: 현재 3카드 세로 나열을 유지하되 각 카드에 정보를 충실히 채움. HotListSection의 정보 밀도를 참고.

```
[카드 1]
┌─────────────────────────────────────────────┐
│ 강세 ●●●●○                    외국인 연속매수  │
│ ┌────┐                                      │
│ │로고│ 삼성전자  82,400원  +1.2%  ~~spk~~   │
│ └────┘                                      │
│ 외국인 5일째 순매수, 3개월 최장 기록            │
│ [외국인 +2,400억] [거래량 1.8x]              │
└─────────────────────────────────────────────┘

[카드 2] (동일 포맷)
[카드 3] (동일 포맷)
```

**장점:**
- 현재 레이아웃 구조 유지 — 최소 변경으로 최대 효과
- 3개 시그널 모두 동등한 정보량 — 사용자가 비교 용이
- HotListSection과 시각적 일관성 유지 (로고 + 이름 + 가격 + 변동률)
- 시그널 수가 1~2개여도 레이아웃 깨지지 않음

**단점:**
- "히어로"라는 이름에 맞지 않는 평등한 구성 — 시각적 임팩트 부족
- 3카드 모두 정보량이 동일하면 "가장 중요한 것"을 놓칠 수 있음
- 토스증권의 "카드 단위 스캔" 패턴과 충돌 — 카드가 너무 많으면 스캔 피로

**구현 난이도:** 하 (기존 구조에 필드 추가)

### C안: 캐러셀 카드형 (Carousel Hero)

**컨셉**: Robinhood "Today's Movers" 스타일의 수평 스와이프 카드. 히어로 영역을 1카드 크기로 고정하고 좌우 스와이프로 3개 시그널 순환.

```
          ← swipe →
┌─────────────────────────────────────────────┐
│                                   1 / 3     │
│ [로고]  삼성전자                              │
│         005930 · KR                         │
│                                             │
│ 82,400원                 +1.2%              │
│ ~~~~~~~~ sparkline (full width) ~~~~~~~~    │
│                                             │
│ 강세 ●●●●○  외국인 연속매수                    │
│ "외국인 5일째 순매수, 3개월 최장 기록"          │
│                                             │
│ ○ ● ○  (페이지 인디케이터)                    │
└─────────────────────────────────────────────┘
```

**장점:**
- 각 시그널에 풀 영역 할당 — 정보 밀도와 여백의 최적 밸런스
- 스와이프 UX는 모바일 네이티브 (토스/Robinhood 패턴)
- full-width sparkline으로 시각적 임팩트 극대화
- 5초 자동 전환 + 수동 스와이프 가능 — 패시브 사용자도 커버

**단점:**
- 한 번에 1개만 보임 — 3개 시그널 비교가 어려움
- 캐러셀 구현 + 터치 이벤트 + 자동 전환 로직 추가 필요
- 데스크톱에서 스와이프가 자연스럽지 않음 (좌우 화살표 필요)
- 자동 전환이 "정보를 놓치게 하는" UX 안티패턴일 수 있음 (Nielsen Norman Group)

**구현 난이도:** 중상 (스와이프 라이브러리 또는 커스텀 구현)

---

### D안: 하이브리드 (1+2 Split) — 권장

**컨셉**: A안의 "1위 강조" + B안의 "정보 충실함"을 결합. 1위는 확장 카드, 2~3위는 인라인 컴팩트 행.

```
[1위 시그널 — 확장 카드]
┌─────────────────────────────────────────────┐
│ [로고] 삼성전자  82,400원  +1.2%  ~~spk~~   │
│                                             │
│ 강세 ●●●●○  외국인 연속매수                    │
│ 외국인 5일째 순매수, 3개월 최장 기록            │
│ [외국인 +2,400억] [거래량 1.8x]              │
└─────────────────────────────────────────────┘

[2위] SK하이닉스  185,000원 +2.8%  강세●●●○○ 거래량 폭증
[3위] 에코프로    67,200원  -3.8%  약세●●●○○ 심리 괴리
```

**장점:**
- 1위에 시각적 위계 부여하면서도 2~3위 정보를 즉시 스캔 가능
- 세로 공간 효율 — A안 대비 30% 절약
- 시그널 1개일 때도 카드 1개로 자연스러움
- 인라인 행은 HotListSection의 HotRow와 동일 패턴 — 학습 비용 제로

**단점:**
- 2~3위 시그널의 meta 데이터(근거)를 표시할 공간이 부족
- 1위와 2~3위의 시각적 격차가 너무 크면 2~3위가 "덤" 느낌

**구현 난이도:** 중하 (기존 HotRow 패턴 재사용 가능)

### 히어로 시그널 — 안별 비교표

| 기준 | A안 (1+2) | B안 (균등3) | C안 (캐러셀) | D안 (하이브리드) |
|------|----------|-----------|------------|--------------|
| 정보 밀도 | 높음 | 높음 | 높음 | 높음 (1위) / 중간 (2~3위) |
| 시각적 임팩트 | 강 | 중 | 강 | 강 |
| 모바일 UX | 중 (스크롤 증가) | 상 | 상 | 상 |
| 데스크톱 UX | 상 | 상 | 중 (스와이프 어색) | 상 |
| 구현 난이도 | 중 | 하 | 중상 | 중하 |
| 빈 상태 처리 | 중 | 상 | 중 | 상 |
| 소형코인 문제 | 해결 (가격+차트로 채움) | 해결 | 해결 | 해결 |

**권장: D안 (하이브리드)** — 정보 밀도 + 시각적 위계 + 구현 효율의 최적 밸런스.

### 빈 상태 / 엣지 케이스 처리

| 상태 | 현재 | 개선안 |
|------|------|-------|
| 시그널 0개 | 스켈레톤 + "시그널 수집 중..." | 시장 요약 카드로 대체 ("오늘 시장은 조용합니다. 주요 지수를 확인해보세요") |
| 시그널 1개 | 카드 1개만 표시 | 1위 확장 카드만 표시 (2~3위 행 숨김) |
| 시그널 2개 | 카드 2개 | 1위 확장 + 2위 인라인 |
| 소형 코인(로고 없음) | 종목명만 큰 글씨 | 이니셜 아바타(`getAvatarBg`) + 가격 + sparkline으로 채움 |
| 장 마감 후 | 오래된 시그널 표시 | "장 마감 후 — 오늘의 주요 시그널" 라벨 + 다음 장 오픈 카운트다운 |

## 2. AI 종목토론 재설계

### 현재 구조 분석

```
AiDebateSection
├── 헤더: "AI 종목토론" (19px bold)
├── 종목 칩: 삼성전자 / SK하이닉스 / NVDA / BTC (+ 관심종목)
├── "AI에게 물어보기" 버튼 → API 호출 (fetchAiDebate)
├── 살 이유 카드 (빨간 배경) — 1줄 텍스트
├── 조심할 이유 카드 (파란 배경) — 1줄 텍스트
├── 확신도 바: 매수 62% ─────── 관망 38%
└── AI 종합 의견: 1줄 텍스트
```

**문제점 상세:**
- 살 이유/조심할 이유가 각 1~2문장 — AI 분석의 근거가 불투명
- "외국인 5일 연속 순매수"라고 하지만, 실제 데이터를 보여주지 않음
- 확신도 바의 숫자가 어떤 기준으로 나왔는지 불명확
- 사용자 참여 요소 전무 — 일방적으로 AI 의견을 받기만 함
- 종목 칩이 하드코딩 4개 + 관심종목 5개 — 발견(discovery) 기능 없음
- 30분 sessionStorage 캐시 — 시장 급변 시 outdated 위험

### API 응답 구조 (fetchAiDebate)

```javascript
{
  messages: [
    { side: 'bull', text: '...' },
    { side: 'bear', text: '...' },
  ],
  verdict: '종합 의견 한 줄',
  confidence: 0.62, // 0~1 매수 확신도
}
```

현재 API가 반환하는 정보만으로는 근거 데이터 표시가 불가 — **백엔드 확장이 필요한 안과 프론트엔드만으로 가능한 안을 분리.**

---

### A안: 근거 데이터 카드형 (Evidence-Based Cards)

**컨셉**: 살 이유/조심할 이유 각각에 근거 데이터 미니카드를 붙임. 백엔드 API 확장 필요.

```
[AI 종목토론 — 삼성전자]

┌─ 살 이유 ──────────────────────────────────┐
│ "외국인 5일 연속 순매수, HBM 수주 확대"       │
│                                             │
│ ┌────────┐ ┌────────┐ ┌────────┐            │
│ │외국인    │ │PBR     │ │뉴스     │            │
│ │+2,400억 │ │1.1x    │ │호재 3건  │            │
│ │5일 연속  │ │역사 저점│ │중립 1건  │            │
│ └────────┘ └────────┘ └────────┘            │
└─────────────────────────────────────────────┘

┌─ 조심할 이유 ──────────────────────────────┐
│ "반도체 업황 피크아웃, 환율 리스크"            │
│                                             │
│ ┌────────┐ ┌────────┐                       │
│ │환율     │ │업종지수  │                       │
│ │1,460원  │ │-2.1%   │                       │
│ │고착 중  │ │3일 하락  │                       │
│ └────────┘ └────────┘                       │
└─────────────────────────────────────────────┘

매수 62% ═══════════════════ 관망 38%

[동의해요 47명] [글쎄요 32명]  ← 투표
```

**장점:**
- 근거 데이터로 AI 분석의 신뢰도 대폭 향상
- 미니카드 패턴은 Seeking Alpha의 Factor Grades와 유사 — 검증된 UX
- 투표로 사용자 참여 유도 (eToro Social Trading 패턴)
- "데이터가 아니라 시그널을 판다"는 서비스 철학에 부합

**단점:**
- 백엔드 API 확장 필수 — fetchAiDebate 응답에 evidence 필드 추가 필요
- 근거 데이터 수집을 위한 추가 API 호출 (외국인 매매, 뉴스 카운트 등)
- 카드가 세로로 길어짐 — 모바일에서 스크롤 과다 위험
- 투표 기능은 서버 사이드 구현 필요 (현재 서버리스 구조에서 복잡)

**구현 난이도:** 상 (백엔드 + 프론트엔드 동시 변경)

- 현재 2줄 요약 구조를 크게 바꾸지 않으면서 가치 추가
- "지난주 결과" 피드백 루프가 재방문 동기 부여

**단점:**
- 투표 집계 서버 구현 필요 (Supabase 또는 별도 API)
- 적중률 계산 로직 구현 필요 — AI 판단 이력 저장 + 결과 대조
- 사용자 수가 적을 때 투표 숫자가 초라해 보일 수 있음
- "나도 매수" 버튼이 투자 조언으로 오해될 법적 리스크

**구현 난이도:** 중상 (투표 서버 + 적중률 로직)

## 3. 종목 티커 이미지 통합 방안

### 현재 상태

`getLogoUrls(item)` 함수가 `src/components/home/utils.js`에 존재하며, 마켓별 fallback 체인을 제공:

| 마켓 | 1차 소스 | 2차 소스 | 3차 소스 | 최종 fallback |
|------|---------|---------|---------|-------------|
| COIN | `item.image` (CoinGecko) | CoinCap assets | cryptocurrency-icons (GitHub) | 이니셜 아바타 (`getAvatarBg`) |
| US | Parqet logos | Clearbit logos | - | 이니셜 아바타 |
| KR | AlphaSquare stock_logo | - | - | 이니셜 아바타 |

### 현재 사용처

| 컴포넌트 | 로고 사용 | 비고 |
|---------|---------|------|
| HotListSection (HotRow) | O | `getLogoUrls` + `onError` fallback |
| NotableMoversSection | O | `getLogoUrls` + `onError` fallback |
| HeroSignalCard | **X** | 종목명 텍스트만 |
| WatchlistMini | **X** | 마켓 배지(KR/US/COIN)만 |
| AiDebateSection | **X** | 텍스트 칩만 |
| SignalBoardWidget | **X** | 시그널 텍스트만 |

### 통합 적용 계획

**원칙: HotRow에서 이미 검증된 `getLogoUrls` + `onError` fallback 패턴을 전 컴포넌트에 동일하게 적용.**

```jsx
// 공통 로고 컴포넌트 추출 (TickerLogo)
function TickerLogo({ item, size = 24 }) {
  const logoUrls = getLogoUrls(item);
  const [logoIdx, setLogoIdx] = useState(0);
  const bg = getAvatarBg(item.symbol);

  if (logoIdx < logoUrls.length) {
    return (
      <img
        src={logoUrls[logoIdx]}
        alt={item.symbol}
        onError={() => setLogoIdx(i => i + 1)}
        className={`w-${size/4} h-${size/4} rounded-full object-contain bg-white border border-[#F2F4F6] p-0.5`}
      />
    );
  }
  return (
    <div
      className={`w-${size/4} h-${size/4} rounded-full flex items-center justify-center text-white text-[9px] font-bold`}
      style={{ background: bg }}
    >
      {(item.symbol || '?').slice(0, 2).toUpperCase()}
    </div>
  );
}
```

**적용 우선순위:**

| 순위 | 컴포넌트 | 이유 |
|------|---------|------|
| 1 | HeroSignalCard | 가장 큰 카드에 로고 없는 것이 "비어보임"의 주범 |
| 2 | AiDebateSection 칩 | 칩 안에 미니 로고(16px)로 시각적 풍부함 |
| 3 | WatchlistMini | 마켓 배지 옆에 로고 추가 (데스크톱만) |
| 4 | SignalBoardWidget | 시그널 행에 미니 로고(20px) |

**주의사항:**
- KR 종목은 AlphaSquare 1개 소스뿐이라 fallback이 이니셜 아바타로 바로 넘어감 — 추가 소스 조사 필요
- 로고 이미지 로딩 실패 시 레이아웃 시프트(CLS) 방지를 위해 고정 크기 컨테이너 필수
- 히어로 카드 로고는 28~32px, 인라인 행 로고는 20~24px, 칩 내 로고는 16px

## 6. 리스크 및 고려사항

1. **법적 리스크**: "매수/관망" 표현이 투자 권유로 해석될 수 있음 → 면책 문구 필수 ("AI 참고 의견이며 투자 판단의 근거가 아닙니다")
2. **API 비용**: sparkline 데이터 조회 증가 → 한투 API 호출량 모니터링 필요
3. **성능**: 히어로 카드에 가격/sparkline 추가 시 allItems 의존성 증가 → useMemo 최적화 필수
4. **소형 코인 문제**: 시그널 강도만으로 히어로 선정 시 유동성 낮은 코인이 올라올 수 있음 → 시가총액/거래량 최소 기준 필터 검토
5. **캐시 전략**: AI 토론 30분 캐시가 시장 급변 시 문제 → 시그널 발생 시 캐시 무효화 로직 검토
