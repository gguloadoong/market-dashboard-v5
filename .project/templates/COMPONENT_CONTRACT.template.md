# [기능명] 컴포넌트 계약

> **이 파일이 [기능명]의 단일 진실 소스다.**
> 컴포넌트를 추가/삭제/변경하기 전에 이 파일을 먼저 읽고,
> 변경 후 이 파일을 반드시 업데이트하라.

---

## 사용법

1. 이 파일을 해당 디렉토리에 복사: `cp COMPONENT_CONTRACT.template.md src/components/[기능]/CONTRACT.md`
2. "현재 활성 렌더 구조"와 "영구 삭제된 컴포넌트" 목록을 채운다
3. CLAUDE.md에 아래 한 줄 추가:
   ```
   **[기능] 컴포넌트를 건드리는 경우**: `src/components/[기능]/CONTRACT.md`를 먼저 읽는다.
   ```

---

## 현재 활성 렌더 구조 ([날짜] 기준)

`src/components/[기능]/index.jsx` 렌더 순서:

```
1. ComponentA  — 역할 설명
2. ComponentB  — 역할 설명
3. ComponentC  — 역할 설명
```

---

## 영구 삭제된 컴포넌트 (절대 재추가 금지)

| 컴포넌트 | 삭제 이유 | 대체 | 삭제 커밋/PR |
|---------|---------|------|------------|
| `OldComponent` | 기능 X로 통합됨 | ComponentB | #000 |

> ⚠️ 이 목록의 파일이 `src/` 어딘가에 남아있으면 즉시 삭제하라.
> **파일 존재 = 살아있는 기능으로 오인.**

---

## 변경 규칙

1. **추가**: 이 파일에 먼저 추가 → index.jsx 수정
2. **삭제**: index.jsx에서 제거 → 파일 삭제 → 이 파일 "영구 삭제" 목록 추가
3. **삭제된 파일 복원 금지**: 목록에 있는 컴포넌트는 새 이름으로만 재구현 가능
