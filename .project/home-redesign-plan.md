## 1. 현재 홈 진단

### 1-1. 현재 렌더 순서 (10개 섹션)

```
 1. MarketPulseWidget       — 시장 지수 + 환율
 2. MarketSentimentWidget   — 시장 심리 (온도계+공포탐욕)
 3. NotableMoversSection    — 주목할 종목 (WHY 카드)
 4. SignalSummaryWidget     — 투자 시그널 (강세/약세 2열)
 5. SeoulForceSection       — 세력 포착
 6. WatchlistWidget         — 관심종목
 7. AiDebateSection         — AI 종목토론
 8. TopMoversWidget         — 급등/급락
 9. NewsFeedWidget          — 투자 뉴스
10. EventTicker             — 경제 이벤트
```

### 1-2. 문제점

| 문제 | 심각도 | 근거 |
|------|--------|------|
| 섹션 10개 — 시각 계층 없이 균등 나열 | 높음 | ADR-014 딥다이브 지적 |
| "시그널이 주인공"인데 4번째에 위치 | 높음 | ADR-016 핵심 정체성 |
| 관심종목이 6번째 — 스크롤 없이 도달 불가 | 중간 | Job 3 충족 미흡 |
| 시장 심리 + 시그널 + 세력 포착이 분산 | 중간 | 동일 "시그널" 범주인데 3개 섹션 |
| 데스크톱 우측 패널(뉴스)과 홈 뉴스(9번) 중복 | 중간 | ADR-003 지적 사항 |
| 모바일에서 10개 섹션 풀스크롤 = 정보 과부하 | 높음 | 8초 KPI 위반 위험 |

### 1-3. 보존 대상 (건드리면 안 되는 것)

- **NotableMovers (주목할 종목)** — WHY 카드, 서비스 핵심 가치

- **우측 뉴스 패널 (BreakingNewsPanel)** — 데스크톱 상시 노출

## 2. 홈 정보 계층 재정의

### 2-1. 3-Layer 모델

```
Layer 1: GLANCE (3초)
  "오늘 시장 어때?" + "지금 가장 강한 시그널은?"
  → 시장 온도 + 히어로 시그널 1~2개

Layer 2: SCAN (10초)
  "내 종목은?" + "뭐가 움직여?"
  → 관심종목 현황 + 주목할 종목 WHY 카드

Layer 3: DIVE (30초)
  "더 자세히 보자"
  → 시그널 전체 + 급등/급락 + AI 토론 + 이벤트
```

### 2-2. 섹션 통합/재배치 방안

| 현재 | 리뉴얼 | 변경 내용 |
|------|--------|----------|
| MarketPulse (1번) | Layer 1 유지 | 축소 — 지수 3개 + 환율 1줄 스트립 |
| MarketSentiment (2번) | Layer 1 통합 | 시그널 스코어보드에 흡수 (독립 섹션 X) |
| NotableMovers (3번) | Layer 2 승격 | WHY 카드 유지, 히어로 1개 + 나머지 리스트 |
| SignalSummary (4번) | Layer 1 히어로 | 최강 시그널 1~2개를 히어로로 승격 |
| SeoulForce (5번) | Layer 3 통합 | SignalSummary 안에 시그널 유형으로 통합 |
| Watchlist (6번) | Layer 2 승격 | 스크롤 없이 보이는 위치로 |
| AiDebate (7번) | Layer 3 유지 | 관심종목 연계 (관심종목 아래 배치) |
| TopMovers (8번) | Layer 3 유지 | 축소 — 탭별 TOP 3으로 경량화 |
| NewsFeed (9번) | 제거 (데스크톱) / Layer 3 (모바일) | 데스크톱은 우측 패널로 충분 |
| EventTicker (10번) | Layer 1 인라인 | MarketPulse 하단에 1줄 롤링 |

## 3. 우측 뉴스 패널 개선안

### 현재 (ADR-004 기반)
- 고래 미리보기 핀 (1건) + 탭 (속보/국내/해외/코인) + 뉴스 피드
- 데스크톱 전용 (`hidden lg:block`), 모바일은 뉴스 탭으로 분리

### 개선 방향
- **데스크톱**: 패널 상단에 "실시간 시그널 피드" 추가 (시간순, 최신 5개)
  - 고래 알림 + 세력 포착 + 강도 3+ 시그널이 시간순으로 흐름
  - 아래에 기존 뉴스 탭 유지
- **모바일**: 뉴스 탭에 "시그널" 서브탭 추가
  - 시그널 + 뉴스 통합 피드 (시간순 인터리빙)

### 4-1. 공통 원칙

- **Breakpoint**: `lg:1024px` (현재와 동일)
- **최대 폭**: `1440px` (현재와 동일)

- 메인 영역: Layer 1 → Layer 2 → Layer 3 수직 나열
- 우측 패널: 시그널 피드 + 뉴스 (sticky, `calc(100vh - 84px)`)
- 홈 NewsFeed 섹션 제거 (우측 패널이 대체)

### 4-3. 모바일 특화

- Layer 3: 접을 수 있는 아코디언 or 더보기 버튼

- EventTicker: MarketPulse 아래 1줄 롤링 (현재와 동일)

## 5. 시그널이 홈에서 주인공이 되는 방법

### 5-1. "시그널 히어로" 개념

현재 SignalSummaryWidget은 강세/약세 리스트로 4번째에 위치한다.
리뉴얼에서는 **가장 강한 시그널 1~2개를 "히어로 카드"로 최상단에 배치**한다.

```
히어로 시그널 카드 구조:
+-------------------------------------------------------+
|  [강세]  삼성전자 — 외국인 5일 연속 매수              |
|  "외국인이 5일째 사고 있어요. 최근 3개월 최장 기록"    |
|                                         [차트 보기 ->] |
+-------------------------------------------------------+
```

- 시그널 강도(strength) 최고값 1~2개 자동 선정
- easyLabel로 제목, 부연 설명 1줄
- 탭 1번으로 ChartSidePanel 오픈
- 시그널 없는 시간대: "시장이 조용합니다" 플레이스홀더

### 5-2. 시그널 스코어보드 (MarketSentiment 통합)

현재 MarketSentimentWidget(온도계 + 공포탐욕)을 시그널 스코어보드로 통합:

```
+-------------------------------------------------------+
|  시장 온도: [====강세====|==약세==]  강세 63%          |
|  공포탐욕: 72 (탐욕)  |  시그널 12개 감지 중          |
+-------------------------------------------------------+
```

- 1줄로 시장 전체 상태 요약
- "시그널 12개 감지 중" 클릭 → Layer 3 시그널 전체 목록으로 스크롤

### 5-3. 시그널 통합 (SeoulForce 흡수)

세력 포착(외국인/기관 연속 매수매도)은 SignalSummary 안에서 시그널 유형 중 하나로 표시.
별도 섹션으로 분리하지 않고, 시그널 리스트에서 "세력 포착" 태그로 구분.

**섹션 구성 (7개, 현재 10개에서 축소)**

| 순서 | 섹션 | Layer | 변경사항 |
|------|------|-------|---------|
| 1 | 시장 스트립 + 이벤트 티커 | L1 | MarketPulse 축소 + EventTicker 인라인 |
| 2 | 시그널 히어로 (1~2개) | L1 | 신규 — SignalSummary에서 최강 추출 |
| 3 | 관심종목 | L2 | 6번 -> 3번 승격 |
| 4 | 주목할 종목 (WHY 카드) | L2 | 유지 |
| 5 | 시그널 전체 (스코어바 + 리스트) | L3 | SignalSummary + SeoulForce + Sentiment 통합 |
| 6 | AI 토론 + 급등/급락 (탭 통합) | L3 | 2개 섹션 -> 1개 탭 위젯 |
| 7 | 뉴스 (모바일만) | L3 | 데스크톱은 우측 패널 |

**장점**
- 시그널이 확실한 주인공 (히어로 카드가 시선 독점)
- 10개 -> 7개 섹션 축소로 정보 과부하 해소
- 3초 안에 "오늘의 핵심" 파악 가능
- 관심종목 스크롤 없이 도달

**단점**
- 시그널이 없는 시간대(장 마감 후)에 히어로 영역이 비어 보일 수 있음
- 시장 지수 정보가 축소되어 지수 중심 사용자 불만 가능
- MarketSentiment 독립 위젯 삭제로 시장 심리 시각화 약화

**참고한 경쟁사/패턴**: Robinhood (히어로 무버), Bloomberg (정보 밀도), 역피라미드 패턴

**섹션 구성 (6개)**

| 순서 | 섹션 | Layer | 변경사항 |
|------|------|-------|---------|
| 1 | 시장 온도 바 (지수 + 심리 통합) | L1 | MarketPulse + Sentiment 1줄 통합 |
| 2 | 관심종목 + 시그널 인라인 | L1+L2 | 관심종목 카드에 해당 종목 시그널 삽입 |
| 3 | 주목할 종목 (WHY 카드) | L2 | 유지 |
| 4 | 시그널 요약 (내 종목 외) | L2 | SignalSummary + SeoulForce 통합 축소 |
| 5 | 급등/급락 + AI 토론 (탭) | L3 | 2개 -> 1개 탭 위젯 |
| 6 | 뉴스 (모바일만) | L3 | 데스크톱은 우측 패널 |

**장점**
- "내 돈"이 가장 먼저 보임 — 심리적 안정감 (토스증권 장점 흡수)
- 시그널이 종목 맥락에 녹아듦 — "이 종목에 무슨 일이?" 즉시 파악
- 6개 섹션으로 최소화 — 정보 과부하 최소
- 관심종목 없는 신규 사용자에게 "종목 추가" 온보딩 자연스러움

**단점**
- 관심종목이 없는 첫 사용자에게 빈 화면 위험 (온보딩 필수)
- 시그널이 관심종목에 종속 — 관심종목 외 시그널 발견성 낮음
- "시그널이 주인공"보다 "관심종목이 주인공" 느낌
- 시장 전체 시그널 파악에 추가 스크롤 필요

**참고한 경쟁사/패턴**: 토스증권 (내 투자 최상단), Robinhood (포트폴리오 퍼스트), 개인화 대시보드 패턴

| 순서 | 섹션 | Layer | 변경사항 |

| 2 | 주목할 종목 (WHY 카드) | L2 | 유지 |

| 4 | 탐색 (급등/급락/AI토론/섹터 탭) | L3 | 4개 -> 1개 탭 위젯 |

- 시그널 히어로 + 관심종목이 같은 시야에 공존
- 5개 섹션으로 최소화 — 가장 간결
- 우측 패널에 시그널+뉴스 통합 피드 — 실시간 감각 극대화
- 탐색 탭에 섹터 추가 — SectorMiniWidget 흡수 가능

**단점**
- 커맨드 센터 블록 내부 복잡도 높음 — 구현 난이도 상
- 관심종목이 "칩" 형태로 축소되어 정보량 감소 (가격만, 수익률 미표시)
- 모바일에서 커맨드 센터가 화면 절반 차지 — 스크롤 시작점 늦어짐
- 데스크톱 2열(히어로+관심종목) 배치가 좁은 화면에서 깨질 위험

**참고한 경쟁사/패턴**: Bloomberg LAUNCHPAD (멀티 위젯 융합), TradingView (커스텀 대시보드), 패턴 D (히어로+리스트 하이브리드)

## 9. 추천 의견 (Strategy)

### 1순위 추천: 시안 A (시그널 히어로)

**근거**:
1. ADR-016 "시그널을 판다"에 가장 충실 — 히어로 카드가 서비스 정체성을 시각적으로 선언
2. 구현 난이도 중간 — 기존 컴포넌트 재배치 + 히어로 추출 로직 추가 수준
3. 신규 사용자 경험 우수 — 관심종목 없어도 시그널이 보임
4. 시그널 없는 시간대 리스크는 "시장이 조용합니다" 플레이스홀더 + 마감 이후 요약으로 해결 가능

### 2순위: 시안 C (커맨드 센터)

**근거**: 정보 밀도와 효율성 최고. 다만 구현 복잡도가 높아 2차 리뉴얼로 진화 가능.

### 보류: 시안 B (내 종목 퍼스트)

**근거**: 토스증권과 동일 방향이 되어 차별화 약화. "시그널 인텔리전스"라는 우리 정체성보다 "포트폴리오 관리 앱"으로 인식될 위험.
