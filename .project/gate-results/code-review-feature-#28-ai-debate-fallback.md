**변경 요약:** 429 전용이던 fallback 조건을 5xx로 확장 + 에러 응답 포맷 리팩터

---

**[STYLE]** 주석의 "5xx(서버 일시 오류)" 표현이 약간 부정확합니다. 501 Not Implemented는 일시적 오류가 아니라 영구적 미지원을 의미합니다. `502/503/504`만 명시하거나 "5xx(서버 측 오류)"로 수정하면 더 정확합니다. 단, Groq API 컨텍스트에서 501이 실제로 반환될 가능성이 낮으므로 실용적으로는 무해합니다.

**[STYLE]** 모든 모델이 5xx를 반환하는 경우(루프 소진 후) 어떤 응답이 반환되는지가 diff에 보이지 않습니다. 이 diff 외부에서 처리되고 있다면 문제없지만, 해당 fallthrough 경로에서도 적절한 에러가 반환되는지 확인이 필요합니다.

---

**긍정적 평가:**
- 핵심 로직 변경(`429 || >= 500`)은 의도가 명확하고 올바릅니다. 5xx는 Groq 서버 측 문제이므로 다음 모델을 시도하는 것이 합리적입니다.
- 4xx(인증/요청 오류)는 재시도해도 동일하게 실패할 것이므로 즉시 반환하는 분기는 정확합니다.
- 포맷 변경은 동작에 영향 없는 순수 스타일 정리입니다.
- 이전 PR(`#28`)의 수정사항과 일관됩니다.
