### (P0-1) 모닝 브리핑 mock 폴백 -- 해결

판정: **해결 완료.** API 미배포 상태에서도 사용자가 브리핑 위젯을 경험할 수 있다. 단, mock 데이터임이 사용자에게 불투명하다는 잔존 이슈가 있다 (아래 잔존 이슈 참조).

### (P0-2) 시그널 엔진 연동 useTopSignals(3) -- 해결

R1 지적: 모닝 브리핑이 지수 + F&G만 표시하고, 계획서가 명시한 "주목: 삼성전자(외국인 매수), 반도체(섹터 상승)" 수준의 시그널 상위 종목이 없다.

수정 내용:
- `MorningBriefing.jsx` 4행: `import { useTopSignals } from '../../hooks/useSignals'`
- 49행: `const topSignals = useTopSignals(3)`
- 162~179행: 시그널 섹션 렌더 블록 추가 -- 구분선 + "시그널" 라벨 + symbol/label/direction(bullish/bearish/neutral) 방향 화살표

Playwright 검증:
- "시그널" 텍스트: 3건 FOUND (위젯 제목, 피드, 브리핑 내 시그널 라벨 포함)
- 실 시그널 데이터: 현 시점 시그널 엔진에 활성 시그널이 없어 `topSignals.length === 0` → 시그널 섹션 미렌더 (조건부 렌더 `{topSignals.length > 0 && ...}`는 정상 동작)

판정: **해결 완료.** 코드 구조는 계획서의 "주목 종목: 전일 시그널 상위 3건" 요구사항을 충족한다. 시그널 엔진이 데이터를 생산하면 브리핑 카드에 자동 표시된다. 시그널이 없을 때 빈 상태인 것은 올바른 동작이다.

### (P1-2) Pull-to-refresh 스피너 UI + 상태별 텍스트 -- 해결

R1 지적: "↻" 문자 하나 + 60px 임계치 구분. 네이티브 앱 수준의 시각적 피드백 미흡.

수정 내용 (index.jsx 250~258행):
- CSS 스피너: `w-5 h-5 border-2 border-[#3182F6] border-t-transparent rounded-full animate-spin`
- 상태별 텍스트 3종:
  - 당기는 중 (pullDistance <= 60): "당겨서 새로고침"
  - 임계치 초과 (pullDistance > 60): "놓으면 새로고침"
  - 새로고침 중 (refreshing): "새로고침 중..."
- opacity 트랜지션: `Math.min(pullDistance / 60, 1)` -- 당기는 거리에 비례하여 서서히 나타남
- height 동적 바인딩: 당기는 중에는 pullDistance, 새로고침 중에는 40px 고정

판정: **해결 완료.** R1의 단순 텍스트에서 시각적 스피너 + 3단계 상태 텍스트 + opacity 트랜지션으로 개선되었다. 네이티브 앱 수준의 스프링 탄성 애니메이션까지는 아니지만, PWA로서 충분한 수준이다.

## 잔존 이슈 (R3 수정 대상)

### P1: PWA Push 알림 미구현 (-15점 영향 추정)

계획서 V3는 "매일 아침 8:50 푸시 알림"을 명시했다. 현재 구현은:
- Vercel Cron 스케줄: 설정 완료 (`50 23 * * *` UTC = KST 08:50)
- 브리핑 API + 위젯 카드: 완성
- 시그널 엔진 연동: 완성

미구현:
- Service Worker Push 구독 (`PushManager.subscribe()`)
- 서버 사이드 Web Push 발송 (web-push 라이브러리 또는 Vercel Edge)
- 사용자 알림 시각 설정 UI

카드 위젯(앱 열어야 보임)과 Push 알림(앱 안 열어도 옴)은 DAU 기여도가 근본적으로 다르다. 서비스 차별화의 핵심 중 하나이다.

### P2: mock 데이터 투명성 (-6점 영향 추정)

현재 mock 폴백 시 사용자가 보는 화면:
- 코스피 2,650, 나스닥 18,200, BTC $131,500,000 -- **하드코딩된 가짜 수치**
- "시장 데이터를 불러오는 중입니다" -- summary만 모호하게 표시

금융 앱에서 가짜 수치를 실제 수치처럼 보여주는 것은 사용자 신뢰를 훼손한다. 최소한:
- "예시 데이터" 배지 표시, 또는
- mock 상태에서는 지수 수치를 "--"로 표시하고 summary만 보여주기, 또는
- "API 연결 대기 중" 명시적 상태 표시

### P2: 브리핑-Market Pulse 지수 중복 (-3점 영향 추정)

브리핑 카드에 코스피/나스닥/BTC 변동률이 표시되고, 바로 아래 Market Pulse에도 동일 지수가 표시된다. 둘 다 화면에 동시 노출되면 정보 중복이다. 브리핑은 "요약 + 시그널"에 집중하고, 상세 지수는 Market Pulse에 위임하는 것이 적절하다.
