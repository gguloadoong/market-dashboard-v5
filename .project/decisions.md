# 아키텍처 결정 기록 (ADR)

## ADR-001: 탭 구조 B안 채택
- **날짜**: 2026-03-17
- **결정**: `[🔥 지금 핫한 것] [🇰🇷 국내] [🇺🇸 해외] [🪙 코인] [📊 ETF]`
- **근거**: 서비스 존재 이유가 "급상승 캐치". 탭 이름이 정체성을 표현해야 함. "홈"은 어디에나 있지만 "지금 핫한 것"은 우리만 할 수 있음.
- **리스크**: 처음 사용자에게 낯설 수 있음. 그러나 익숙함보다 빠른 이해가 목적.

## ADR-002: CDS 한국 색상 오버라이드
- **날짜**: 2026-03-17
- **결정**: Coinbase Design System 구조/토큰 시스템 차용 + 색상값만 한국 증권 컨벤션으로 오버라이드
- **근거**: CDS는 국제 기준(green=상승) 사용. 한국 투자자에게 빨강=상승, 파랑=하락이 체화됨. 색상 변경 시 신뢰 깨짐.
- **차용**: 타이포 스케일, 8px 간격, border-radius 체계, elevation, DataCard/Table 구조, breakpoints
- **오버라이드**: fgPositive → `#F04452`, fgNegative → `#1764ED`

## ADR-003: 홈 BLOCK 3 콘텐츠 교체
- **날짜**: 2026-03-17
- **결정**: "속보 뉴스" → "핫한 종목 인사이트"로 교체
- **근거**: 데스크탑에서 홈 뉴스 + 우측 패널 뉴스 중복. 급등 종목에 관련 뉴스를 매칭하면 "맥락 연결" 문제 해소.
- **규칙**: 매칭 뉴스 없으면 섹션 숨김

## ADR-004: 우측 패널 → 시장 인텔리전스 패널

## ADR-005: 차별화 포인트 3가지
- **날짜**: 2026-03-17
- **결정**:

  3. **투명한 실시간성** — 데이터 지연/정확도 숨기지 않음

## ADR-008: ETF 가격 조회 → Vercel Function 프록시 전환
- **날짜**: 2026-03-19
- **결정**: `fetchEtfPricesBatch` 클라이언트 직접 Yahoo 호출 → `api/etf-prices.js` Vercel Function 프록시로 전환
- **근거**: 클라이언트 직접 호출 시 레이트리밋·IP 차단 리스크, CORS 에러 무피드백, 소형 ETF(ETHU/BITX 등) 데이터 불안정
- **효과**: 서버 캐싱 가능, 레이트리밋 서버 단 관리, fallback 로직 중앙화

## ADR-010: Phase 4 신규 기능 후보 3개 선정 (research-review)
- **날짜**: 2026-03-25
- **담당**: 이지원 (Strategy) + 이준혁 (CPO)
- **컨텍스트**: Phase 1~3 백로그 전체 완료. P0/P1 소진 → 전략 리서치 단계 진입
- **경쟁사 갭 분석**:
  - 토스증권: 단순 알림만 있고 조건 설정 불가, 멀티마켓 없음
  - 업비트: 코인 전용, 주식 없음
  - 키움HTS: 수익률 트래킹 있지만 복잡, 모바일 UX 열악
- **결정**: 아래 3개를 P2로 등록
  1. P2-4 가격·거래량 조건 알림 (PWA Push) — DAU 직결, 조건부 알림으로 차별화
  2. P2-5 매수가 입력 → 평가손익 트래킹 — Job 3 강화, localStorage로 계정 불필요
  3. P2-6 뉴스 임팩트 스코어 (규칙 기반) — "왜 지금?" 다음 단계, AI 없이 구현 가능
- **우선순위 근거**: P2-5 → P2-6 → P2-4 순 (P2-5는 구현 난이도 낮고 즉시 가치 제공, P2-4는 iOS PWA 제약 확인 필요)
