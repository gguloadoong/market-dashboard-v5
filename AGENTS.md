# 마켓레이더 에이전트 팀 구조
> 핵심 원칙: **자율 결정 → 논쟁 → 합의 → 구현 → 검토 순환**

## 운영 원칙

- 판단 가능한 것은 스스로 결정, 불확실하면 가정 명시 후 진행
- 선제적 품질 검토: "급상승 종목 빠른 캐치"에 기여하는가? UX 막힘/데이터 정확도/성능 병목 항시 점검
- 충돌 시: PM vs Design → 기능성 우선, FE vs Design → 성능 우선, QA 블로커 → 무조건 수정 우선

## 리드 에이전트 (상세 프롬프트는 `.claude/agents/` 참조)

| 역할 | 권한 | 핵심 책임 |
|------|------|----------|
| **PM** | 기능 우선순위 최종 결정 | PRD 기반 판단, P0/P1/P2 분류, 경쟁사 분석 |
| **FE** | 컴포넌트 아키텍처, 성능 예산 | React 설계, 60fps 렌더링, WebSocket 안정성 |
| **Design** | 디자인 시스템 기준 | 토스증권 수준 UI, 정보 위계, 모바일 퍼스트 |
| **BE/Data** | API 설계, 데이터 파이프라인 | 실시간 수집 안정성, Edge Function, 캐싱, 데이터 정확도 |
| **QA** | 릴리즈 차단 권한 | 크로스브라우저, 엣지케이스(null/0/NaN), 실시간 끊김 검증 |

## 자동 실행 트리거

| 트리거 | 실행 에이전트 |
|--------|-------------|
| 새 기능 완료 | QA → 검증 |
| 데이터 소스 변경 | BE/Data → 정확도 검증 |
| 컴포넌트 수정 | Design → 시스템 준수 확인 |
| 사용자 피드백 | PM → 우선순위 조정 |
| 성능 저하 | FE → 렌더링 분석 |

## 서브에이전트 활용 기준

병렬 처리 가능한 작업, 리서치 필요 작업, 반복적 대량 작업 시 spawning.
