# request-to-ceo — CEO 헬프 데스크

**트리거:** 외부 블로커(API 키 발급, 환경변수 설정, 결제 권한, 사업 피벗 등)가 발생해 작업이 중단될 경우 `autopilot-loop`가 자동으로 호출합니다.

---

## 팀이 자율 판단하는 것 (CEO에게 묻지 않음)

| 묻지 않는 것 | 이유 |
|------------|------|
| 기술 선택 (라이브러리, 아키텍처) | 팀 전문 영역 |
| 구현 방법 | 팀 전문 영역 |
| 버그 수정 | 팀 전문 영역 |
| 디자인 디테일 | 최유나(Designer) 판단 |
| API fallback 전략 | 김민준(BE) 판단 |
| 테스트 전략 | 장성민(QA) 판단 |

## CEO에게만 묻는 것

| 요청 항목 | 예시 |
|----------|------|
| 외부 서비스 결제 | Vercel Pro 업그레이드, 유료 API 구독 |
| 환경변수/API 키 발급 | 한투 Open API 키, 새 서비스 키 |
| 사업 방향 피벗 | 투자 도구 → 커뮤니티 전환 등 |
| 법적/규제 판단 | 금융 정보 제공 범위, 저작권 |
| 최종 프로덕션 배포 승인 | 대규모 리디자인 배포 전 |

---

## BLUF 보고 양식

이준혁(CPO)이 아래 양식으로 보고한다:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 CEO 보고 요청 — [날짜]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 요청 요약 (한 줄)
   → [무엇이 필요한지]

2. 필요한 이유
   → [왜 이게 없으면 작업이 막히는지]

3. 팀의 사전 검토 내용
   → [대안을 시도했는가? 왜 실패했는가?]

4. CEO 액션 아이템
   → [복붙 가능한 터미널 명령어 또는 URL]
   예: vercel env add VITE_KIS_API_KEY production

⏳ 응답 대기 중. 블로커가 걸린 태스크: [태스크명] → 대기 상태
다른 P1/P2 태스크로 계속 진행합니다.
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

보고 후 CEO 응답을 기다리되, 블로커가 걸린 태스크만 '대기' 상태로 표시하고 **다른 태스크로 계속 진행**한다. 절대 멈추지 않는다.

**[체이닝]** 완료 후 즉시 `autopilot-loop`를 호출하라.
