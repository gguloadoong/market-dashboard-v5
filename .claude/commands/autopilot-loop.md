# autopilot-loop — 마켓레이더 자율 운영 루프

> **핵심 원칙:** 이 루프는 사용자 입력을 기다리지 않는다. 모든 판단과 실행을 자율적으로 수행하라. 이 루프는 절대 멈추지 않는다.

---

## 실행 흐름 (무한 루프)

### 1. 상태 파악

다음 문서를 읽어 현재 상태를 파악하라:
- `.project/backlog.md` — 잔여 태스크, P0/P1/P2 항목
- `.project/decisions.md` — ADR, 이전 판단 기록
- `.project/roadmap.md` — Phase 진행 상황
- `.project/quality-baseline.md` — 현재 품질 기준선
- `git log --oneline -10` — 최근 커밋 상태
- `gh issue list --state open` — 미완료 Issue

---

### 2. ⛔ 프로세스 검증 게이트 (HARD GATE)

**통과 전 다음 단계 진입 불가.** 아래 5가지를 순서대로 점검하라.

- **① Git 소급 정리:** `git log main --oneline -20` 확인. `feat:`/`fix:` 커밋이 PR 없이 main에 있는가? → 소급 Issue 생성 + close 처리
- **② 브랜치 없는 Issue:** `gh issue list --state open` 확인. 대응 브랜치 없는 Issue → 브랜치 생성
- **③ 빌드 상태:** `npm run build` 에러 0개 유지. 실패 시 P0 Issue + 즉시 수정
- **④ 팀 구성 검토:** backlog에 기존 팀으로 해결 불가한 태스크 있는가? → create-agent 검토
- **⑤ 품질 래칫 확인:** `.project/quality-baseline.md` 기준 위반 항목 있는가? → P0 Issue 생성, 모든 신규 작업 중단, 복구 후 재개

---

### 3. 자동 스케줄링

아래 조건을 평가하고 해당 커맨드를 실행하라:

| 조건 | 실행 커맨드 |
|------|------------|
| 매일 아침 (첫 루프) | `standup` |
| 스프린트 시작 | `sprint` |
| 스프린트 종료 | `retro` |
| design-system.md 변경됨 | `design-review` |
| PR 리뷰 통과 후 | `ai-review` |
| 역할 부족 | `create-agent` |
| 외부 리소스/결제 필요 | `request-to-ceo` |
| backlog P0/P1 비어있음 | `research-review` |
| roadmap 큰 수정 필요 | `planning-review` |
| 새 기능 병합 후 | `qa-browser` |
| 위험 명령어 감지 | `safety-guardrails` |

---

### 4. 우선순위 결정

이준혁(CPO)과 이지원(Strategy)이 backlog P0 → P1 → P2 순서로 우선순위를 결정하고 담당자를 배정하라.

---

### 5. 작업 실행

에이전트는 Edit, Bash, Fetch 등 도구를 활용하여 코드/문서를 작성하라.

**반드시 Issue → Branch → Code → PR 워크플로우를 따른다:**
```bash
gh issue create --title "feat: ..." --label "enhancement,ai-generated"
git checkout -b feature/#이슈번호-설명
# 작업
git commit -m "feat: 설명 (#이슈번호)"
gh pr create --title "..." --body "Closes #이슈번호\n\n---\n🤖 Generated by Claude Code [claude-sonnet-4-6]"
```

---

### 6. Fix-Loop Circuit Breaker (수정 무한루프 차단)

**같은 에러에 대해 같은 수정을 3회 시도했는데도 해결되지 않으면 즉시 STOP.**
1. 시도한 수정과 에러 내용을 `.project/decisions.md`에 진단 기록
2. CEO에게 에스컬레이션: "동일 에러 3회 반복 — 다른 접근법이 필요합니다"
3. 해당 태스크를 backlog에 `blocked` 태그로 표시하고 다음 태스크로 이동

---

### 7. 문제 기록

버그, 설계 누락, 성능 이슈, API 불안정은 즉시:
- `.project/backlog.md`에 적절한 우선순위로 등록
- `gh issue create`로 GitHub Issue 생성

---

### 8. 자율 진화 엔진 (루프 유지)

Step 1로 돌아가라. 잔여 태스크가 없어도 아래 자율 활동을 수행한다.

#### Level 1 — 기술 품질 (매일)
- **빌드:** `npm run build` 에러 0개. 실패 즉시 P0
- **번들 사이즈:** 전일 대비 +5% 증가 시 원인 분석 → dynamic import / tree shaking
- **보안:** `npm audit` critical/high 0개. 발견 즉시 P0 패치
- **API 안정성:** `api/` 엔드포인트 fallback 체인 점검

#### Level 2 — UX/데이터 신뢰성 (주 1회, 월요일)
- 실시간 데이터 정확도 감사 (changePct, sparkline, change)
- UI 감사: 빈 상태, 로딩, 에러 화면 점검
- 경쟁사(토스증권, 키움, 업비트) UI 변경 스캔
- 개선 항목 → backlog [UX] 태그로 등록

#### Level 3 — 전략 탐색 (격주, 금요일)
- 경쟁사 신규 기능 심층 분석
- 신규 기능 제안 3개 → `.project/backlog.md` P2 등록
- 데이터 소스 대안 조사 (Yahoo/Stooq/KIS 안정성)
- 사업 방향 피벗 수준이면 `request-to-ceo`, 나머지는 backlog P2

#### Level 4 — 프로세스 개선 (retro 직후)
- Issue 완료율, PR 사이클 타임 메타 분석
- 병목 해결: 사이클 타임 2일 초과 → ADR 기록 (`.project/decisions.md`)
- `.project/quality-baseline.md` 갱신 — 달성한 최고 품질로 상향 (래칫 효과)

---
