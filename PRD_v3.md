## 1. 제품 개요

### 핵심 한 줄
**국장·미장·코인의 급상승 종목을 누구보다 빠르게 캐치하는 실시간 트레이딩 모니터링 웹앱**

### 만든 이유
기존에는 국장은 네이버 금융, 미장은 야후 파이낸스, 코인은 업비트를 각각 켜놓고 봐야 했다.
이 앱 하나로 세 시장을 동시에 보면서 급등 종목을 놓치지 않는 게 목적이다.

### 목표 사용자
- 국장·미장·코인을 동시에 모니터링하는 개인 투자자
- 급등 종목을 빠르게 캐치해 단기 매매 기회를 찾는 트레이더
- ETF 중심으로 시장 흐름을 파악하는 투자자

### 핵심 성공 지표
- 급상승 종목을 경쟁 서비스보다 빠르게 시각적으로 인지할 수 있는가
- 하나의 화면에서 세 시장을 동시에 파악할 수 있는가
- 종목 클릭 → 차트·뉴스 확인까지 3초 이내인가

## 4. 컴포넌트 상세 스펙

### 4.1 급상승 티커 배너
- 조건: 등락률 **+3% 이상** 종목만
- 형식: `삼성전자 +5.23% ▲` 간격 두고 반복
- 애니메이션: 좌→우 마퀴, 속도 일정하게
- 클릭: 해당 종목 카드로 스크롤 이동
- 비어있을 때: 배너 숨김 처리

### 4.2 지수 요약바
| 탭 | 표시 지수 |
|----|---------|
| 전체 | KOSPI, NASDAQ, BTC |
| 국장 | KOSPI, KOSDAQ |
| 미장 | NASDAQ, S&P500, DOW |
| ETF | 숨김 |
| 코인 | BTC 도미넌스, 전체 시총 |

## 6. 정렬 및 필터

### 정렬
- 등락률 높은 순 ← **기본값**
- 등락률 낮은 순
- 거래량 높은 순
- 시가총액 높은 순

### 필터
- 급상승만 (+3% 이상)
- 급하락만 (-3% 이하)
- 거래량 급증 (평균 대비 200% 이상)

## 7. API 연동 상세

### 7.1 코인 — CoinGecko API v3
| 항목 | 내용 |
|------|------|
| 인증 | 불필요 (무료) |
| 엔드포인트 | `GET /coins/markets?vs_currency=krw` |
| 갱신 주기 | 10초 |
| 스파크라인 | `?sparkline=true` 파라미터로 7일치 포함 |
| 차트 데이터 | `GET /coins/{id}/market_chart?days=N` |

### 7.2 해외주식 — Yahoo Finance
| 항목 | 내용 |
|------|------|
| 인증 | 불필요 (비공식 무료) |
| 시세 | `https://query1.finance.yahoo.com/v8/finance/quote` |
| 차트 | `https://query1.finance.yahoo.com/v8/finance/chart/{symbol}` |
| 갱신 주기 | 30초 (장 중), 5분 (장 외) |

### 7.3 국내주식 — 한국투자증권 OpenAPI
| 항목 | 내용 |
|------|------|
| 인증 | APP_KEY + APP_SECRET → Access Token 발급 |
| 시세 | 국내주식 현재가 API |
| 갱신 주기 | 30초 (평일 09:00~15:30) |
| 환경변수 | `VITE_KIS_APP_KEY`, `VITE_KIS_APP_SECRET` |

### 7.4 뉴스 — NewsAPI.org
| 항목 | 내용 |
|------|------|
| 인증 | API Key 필요 (무료 플랜 100req/day) |
| 엔드포인트 | `GET /v2/everything?q={종목명}` |
| 표시 개수 | 최신 5개 |
| 갱신 주기 | 5분 |
| 속보 기준 | 1시간 이내 → 🔴 배지 |
| 환경변수 | `VITE_NEWS_API_KEY` |

### 7.5 환율 — Yahoo Finance
- 심볼: `KRW=X`
- 갱신 주기: 1분
- 용도: 미장 종목 원화 환산

### 7.6 캐싱 전략
| 데이터 | 캐시 시간 |
|--------|---------|
| 코인 시세 | 10초 |
| 주식 시세 | 30초 |
| 뉴스 | 5분 |
| 환율 | 1분 |
| 차트 (1일) | 1분 |
| 차트 (1주~) | 10분 |

## 10. 개발 순서 (권장)

```

  → 상세 모달 완성

Phase 6 — 배포
  → Vercel 연동, 환경변수 설정
```

## 11. 배포

### 로컬 개발
```bash
cd ~/Documents/market-dashboard-v2
npm run dev
# → http://localhost:5173
```

### Vercel 배포 (추천)
```bash
npm install -g vercel
vercel
```
- GitHub 연동 시 `git push` 만 하면 자동 배포
- Vercel 대시보드 → Settings → Environment Variables 에 .env 값 동일하게 입력

### 전체 흐름
```
로컬 작업 → npm run dev로 확인
→ git commit & push
→ Vercel 자동 빌드/배포
→ https://your-app.vercel.app 공유
```
